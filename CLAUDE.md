# CLAUDE.md — AX 인재전쟁 예선 워크스페이스

## 최우선 원칙 (모든 세션·서브에이전트에 적용)

1. **목적은 해커톤 문제풀이다.** Codex Plugin과 SKILL.md를 만들어 제출하는 것이 주된 목적. 다른 모든 산출물(리서치·문서)은 이를 위한 수단이다.
2. **스코프**: 사람이 ~2주 걸릴 분량의 과제를 해결하는 수준. 사업 맥락에 맞는 클러치 피처 하나에 집중. 범용 프레임워크·과잉 설계 금지. (주의: 제출 마감은 2026-07-10 23:59:59 — "2주"는 플러그인이 겨냥하는 문제의 크기이지 작업 일정이 아니다.)
3. **시장조사·문제조사는 수단이다.** 발견이 플러그인 설계 결정으로 이어지지 않으면 그 리서치는 중단한다.
4. **회사 간 절대 분리.** 제출물은 회사별 독립. 리서치·코드·로그·문서가 회사 간에 섞이면 안 된다. 회사 작업은 반드시 해당 회사 폴더에서 세션을 시작하고(`cd <회사폴더> && claude`), 다른 회사 폴더는 읽지도 않는다. 루트 세션에서는 회사별 리서치·구현을 하지 않는다.
5. **로그 불가침.** `logs/` 아래 파일은 절대 편집·발췌·삭제 금지 — 실격 사유. 훅이 자동 생성하며, 잘못 뽑힌 로그도 지우지 않는다.
6. **질문 5문항 초안은 각 회사 폴더의 `question.md`로 관리.** 자수 제한(문항 3은 1000자, 나머지 800자)과 출처 URL(공개 접근 가능)을 준수한다.

## 규정 요약 (전체는 submission-spec.md — 확정/해석 구분 포함)

- 마감: **2026-07-10 (금) 23:59:59**. 마감 후 제출 불가, 마감 전에는 제출 후에도 재제출 가능 → 최소 버전을 일찍 제출하고 개선한다.
- 제출물(회사별): submission.zip (≤100MB) = `src/`(플러그인 루트, `.codex-plugin/plugin.json` 필수) + `README.md` + `logs/`.
- 플러그인은 **평가 시 직접 구동 가능**해야 한다.
- 근거는 **공개 자료 + URL**만 인정. 비공개 내부 정보·사적 경험·출처 없는 숫자 불인정.
- 로그·플러그인·question.md(5문항) 사이 **정합성**이 채점 대상이다.
- AI 채팅에 API 키·비밀번호·토큰 등 비밀정보 입력 금지 (로그에 남는다).

## 저장소 구조

```
ax-hackathon/                  # 공용 워크스페이스 (이 repo, 회사 내용 금지)
├── CLAUDE.md                  # 이 파일 — 최우선 원칙
├── README.md                  # 저장소 안내
├── WORKFLOW.md                # 작업 파이프라인 + 제출 체크리스트
├── RESEARCH_PLAYBOOK.md       # 리서치 방법·출처 규칙
├── submission-spec.md         # 제출 규정 전체 정리 (원문 검증 완료)
├── logs/                      # 루트 세션 로그 (훅 자동 생성)
├── channeltalk/               # 채널톡 제출 워크스페이스
├── kakaopay-securities/       # 카카오페이증권 제출 워크스페이스
└── samil-pwc/                 # 삼일PwC 제출 워크스페이스
```

각 회사 폴더는 자체 CLAUDE.md·README.md·question.md를 가지며, 최종적으로 그 폴더에서 submission.zip을 만든다.
