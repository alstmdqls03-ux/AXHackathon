# haenggan-audit — 결산 데이터 "행간 감사" Codex 플러그인 (삼일PwC 과제)

## 개요

**결산 데이터 패키지(시산표·전표·월별 시계열·채권연령·원가배부표)를 받아, 회계 전문가의 가설 라이브러리 3종을 따라 결정론 교차 검산과 AI 추론으로 '행간에 숨은 문제'를 찾아내고, 발견마다 원본 근거(전표ID·계정·월·제품)를 인용한 경영진 SOP 보고서를 생성하는 Codex 플러그인.**

쓰는 사람: 삼일PwC의 회계사·컨설턴트(감사·자문 현장에서 고객 결산 패키지를 받았을 때)와 그 고객사의 CFO·재무팀·경영진. 개별 시스템 보고서는 다 정상처럼 보이는데 "숨은 문제가 없는가"를 결산 마감·이사회 보고 전에 확인해야 할 때 — 시트·계정·기간을 가로지르는 교차 검토는 사람이 하면 수 주가 걸려 실제로는 사후(정정 공시·감리 지적)에야 드러나는 병목을 대체한다.

가설 라이브러리 (독립 모듈):
- **H1 회계처리 오류** — 비용의 자산화, 기말 몰아치기, 승인 통제 우회, 계정 급변
- **H2 현금흐름 이상** — 흑자 속 영업CF 악화, 회수기간(DSO) 급등, 분기말 밀어내기, 거래처 편중
- **H3 원가배분 왜곡** — 배부 기준 왜곡으로 흑자로 보이는 손실 제품

## 설치·실행 (심사자용)

요구사항: Codex CLI + `python3` (표준 라이브러리만 사용).

1. 로컬 marketplace 폴더를 만들고 `src/`를 플러그인으로 복사:
   ```bash
   mkdir -p haenggan-marketplace/plugins haenggan-marketplace/.agents/plugins
   cp -r src haenggan-marketplace/plugins/haenggan-audit
   cat > haenggan-marketplace/.agents/plugins/marketplace.json <<'EOF'
   {
     "name": "samil-local-plugins",
     "plugins": [
       {
         "name": "haenggan-audit",
         "source": { "source": "local", "path": "./plugins/haenggan-audit" }
       }
     ]
   }
   EOF
   ```
2. marketplace 등록 후 설치:
   ```bash
   codex plugin marketplace add ./haenggan-marketplace
   codex plugin add haenggan-audit@samil-local-plugins
   ```
   (`codex plugin list`로 `installed, enabled` 확인. 대화형을 선호하면 Codex 실행 → `/plugins`에서 설치해도 된다.)
3. 프롬프트 입력:
   ```
   haenggan-audit 스킬로 (플러그인의) data/demo 를 행간 감사해서 report.md를 만들어줘
   ```
   (임의의 결산 데이터 폴더를 지정해도 된다 — 파일 계약은 SKILL.md 참조)

## 작동 방식 (문항 3과 동일 내용)

**절차** — ① 데이터 인벤토리·스키마 대조 → ② 결정론 체커 실행(`scripts/checks.py`: 시산표 산술 검산, 기말 대형 전표·작성자=승인자·수선성 자산화 패턴 스캔, 이익-CF 괴리·DSO 추세·분기말 스파이크·채권 편중 계산, 대안 드라이버 원가 재배부) → 이상 후보 JSON 산출 → ③ 에이전트가 가설 모듈 3종을 로드해 후보를 원본과 대조·검증 → ④ 오탐 기각(기각도 기록) + 신호 연결(예: 수선비 급감 × 유형자산 대형 전표 = 자본화 의심 — 이 교차가 '행간') → ⑤ 심각도·확신도 판정 → ⑥ SOP 보고서 생성.

**지식** — 가설 모듈에 내장: K-IFRS 1016 후속원가(수선비 자본화 요건), 기간 귀속, 승인 분리 통제, 이익의 질(발생액-현금 괴리), DSO·채널 스터핑 패턴, 원가 드라이버 인과성(관리회계 원칙).

**판단 기준** — 심각도(高: 손익·의사결정 반전/기준 위반 소지), 확신도(高: 데이터만으로 입증 / 中 이하: '다음 확인 절차' 필수 첨부), 인용 규칙(모든 발견은 전표ID·계정·월·제품 좌표 인용 — 인용 불가 발견은 출력 금지).

**분업 원칙** — 산술은 결정론 스크립트(재현 가능), 판단(기각·연결·기준 적용·권고)은 AI. "자료를 정리해 보여주는 것"이 아니라 추론 방향(가설)을 받아 추론하고 근거를 제시하는 구조다.

**정보 부족·실패 시 동작** — ① 파일 누락: 해당 가설군만 "검토 불가(사유)" 명시, 나머지 정상 수행(모듈 독립) ② 스키마 불일치: 동의어 매핑 → 1회 질문 → 불가 시 해당 가설군 제외 ③ 발견 0건: 억지 발견 금지, "검토 가설 × 통과 근거" 부록 보고 ④ 체커 실행 실패: 가설 모듈의 체크 정의를 따라 에이전트가 직접 계산하는 수동 모드 폴백(보고서에 명시), 재검산 불일치 시 확신도 강등 ⑤ 전표가 발췌본이면 "제공 범위 내"로 결론 한정.

## 검증

- **정상**: 데모 패키지 실행 → 체커가 이상 후보 19건 산출 → 에이전트가 오탐(정기 감가상각 분개 등)을 기각하고 3건의 발견 확정: F-01 수선비 8.4억(원시값 840, 단위 백만원) 자본화 의심(전표 #4712 — 자본화 패턴·기말 대형·작성자=승인자 3중 신호 × 수선비 전기比 -78%), F-02 흑자 속 영업CF 3개월 연속 음수(DSO 49→87일, 12월 매출 스파이크 1.84배, 대한상사 편중 46%·61일+ 연체 42%), F-03 제품 B 마진 +12.0% → 기계시간 기준 재배부 시 -3.0% 부호 반전.
- **예외**: `monthly_series.csv` 제거 후 실행 → 체커·보고서 모두 H2만 "검토 불가(필수 파일 부재)" 처리, H1·H3 정상 (exit 0) — 모듈 독립성 확인.
