## 공통 배경 (요약)

출제 과제는 **"Trusted CEO Agent" — 기업에 산재된 데이터를 총망라해 기업의 고민(숨은 문제)을 뽑아내는 에이전트**이며 [S01, 01:32–01:44], 출제자는 뽑아낼 것의 예시로 ① 틀린 회계처리 ② 현금흐름 문제 ③ 원가배분 왜곡(숨은 손실 제품)을 열거했다 [S01, 01:59–02:08]. 본질은 "**데이터의 행간** 속에 묻힌 요소 찾기" [S01, 02:08–02:13]. 평가는 구현보다 **기획·의도**, 단순 정리 감점, **가설→추론→근거의 자연어 제시**, **SOP에 맞닿은 설명 구조** [S01, 02:16–04:06]. 회사 배경: AX Node의 실제 제품 방향(경영 인텔리전스 에이전트, 디지털 솔루션 매출 200억)과 일치 [S02·S07·S11] — 상세는 `company.md`.

## 후보 A — 결산 데이터 속 "틀린 회계처리" 사전 탐지

**문제 정의**: 기업 재무팀·결산 담당자(그리고 이를 검토하는 회계법인)는 결산 시점에 수만 건의 전표·원장·시산표에서 잘못된 회계처리(계정 오분류, 기간 귀속 오류, 비정상 분개)를 **수작업 표본 검토**로만 잡는다. 전수 교차 검토는 공수상 불가능해 오류는 공시 후 정정·감리 지적으로야 드러난다.

**근거**:
- 힌트 영상: "틀린 회계 처리를 찾는다도 될 수 있고요" [S01, 01:59–02:01] — 출제자가 든 **첫 번째** 예시
- 상장사 131사(5.1%)가 매년 감사보고서 정정, 총 379회, **재무제표 본문 정정이 72%** [S17]
- 금감원 감리 표본 자체가 "분식위험지표가 높은 회사·회계오류 수정 회사" 기준 — 사전 자가 점검 수요의 제도적 방증 [S19]
- 실무 부하: 결산·감사 시즌 과로 이슈 [S23], 삼일 스스로 반복 업무 연 20만 시간 자동화 [S06]

**Opportunity Sizing**:
- 사후 발견 규모: 131사 ÷ 2,569 상장사 = **5.1%/년** (계산식, S17). 정정 379회 중 본문 273회(72%) — (추정) 본문 정정 유형은 결산 단계 교차 검증으로 탐지 가능한 범주(계정·금액 오류)가 대부분이라는 해석.
- (추정) 예방 가치 = 정정 공시·재감사 대응 공수 + 감리 표본 선정 리스크 회피 — 공개된 단가 통계가 없어 수치화하지 않음.

**플러그인 적합성**: 사람 ~2주(결산 검토 슬라이스) ✓ / 합성 데이터(시산표+원장) 설계 최용이, 근거 인용(전표 ID·계정) 데모가 가장 선명 ✓ / 마감 내 리스크 최소 ✓. 단, 과제 예시 3종 중 1종만 커버.

## 후보 B — 현금흐름 행간의 유동성 위험 조기 경보

**문제 정의**: CEO·이사회는 손익 중심 보고를 받는다. 손익은 흑자인데 영업현금흐름·운전자본(채권 회수, 재고 회전, 만기 구조)이 악화되는 "행간의 신호"는 시스템 어디에도 한 화면으로 존재하지 않아, 계속기업 리스크가 감사보고서 강조사항으로 붙고서야 인지된다.

**근거**:
- 힌트 영상: "현금 흐름상에 뭔가 문제점을 찾는다일 수도 있고요" [S01, 02:01–02:03]
- 2025 회계연도 상장사 2,702곳 중 **계속기업 불확실성 '주의' 66곳(2.4%)** (계산식: 66÷2,702, S20)
- CEO의 AI 성과·신뢰 체감 부족(12~14%) — 경영진용 신호 탐지의 신뢰 격차 [S05·S14]

**Opportunity Sizing**:
- 경보 대상 풀: 상장사만 66곳/년 [S20]. (추정) 비상장 포함 시 수 배 — 공개 통계 미확보로 수치 주장하지 않음.
- (추정) 조기 경보 1건의 가치 = 자금 조달·구조조정 리드타임 확보 — 정량화 불가, 수치 주장하지 않음.

**플러그인 적합성**: 사람 ~2주(자금·신용 분석 슬라이스) ✓ / 데모 가능하나 **시계열 데이터**가 필요하고 판단이 거시적이어서 "근거 = 특정 전표"식의 선명한 인용 데모가 후보 A보다 약함 △ / 마감 내 가능 ✓. 과제 예시 1종 커버.

## 후보 C — 결산 패키지 "행간 감사" 에이전트 (가설 라이브러리 기반 통합) ★추천

**문제 정의**: CEO는 ERP·회계·판매·자금 등 **시스템별 보고**는 다 받지만, 시스템·계정·기간을 **가로질러 교차 대조해야만 보이는 문제**(틀린 회계처리 / 손익-현금흐름 괴리 / 원가배분 왜곡으로 숨은 손실 제품)는 아무도 못 읽는다. 개별 숫자는 전부 정상처럼 보이기 때문이다. 사람이 하려면 회계사급 전문가가 결산 패키지 전체를 수 주간 전수 교차 검토해야 하고, 그래서 실제로는 사후(정정 공시·감리 지적·강조사항)에야 드러난다.

**근거** (출제자 힌트가 이 후보의 정의 그 자체):
- "기업에 산재되어 있는 굉장히 많은 데이터들을 **총망라해서 기업의 고민을 뽑아내는** 에이전트" [S01, 01:36–01:44] — 과제의 원문 정의
- 예시 3종이 하나의 문장 흐름에서 병렬 열거됨("~도 될 수 있고요 / ~일 수도 있고요") [S01, 01:59–02:08] → 세 증상을 관통하는 **가설 목록**이 과제의 실체
- "종합적인 행간을 읽어서" [S01, 01:51], "데이터의 행간 속에 묻어 있는 요소" [S01, 02:10]
- "문제가 완전하게 디파인되지 않았을 수도" + "의도를 넣어 놓은 문제점들" [S01, 02:51–03:04] → 미지의 문제를 가설 구동으로 수색하는 설계 필요
- 사후 발견의 공개 실태: 정정 5.1%/년 [S17] + 계속기업 주의 66곳 [S20] + 감리 표본 기준 [S19]
- 회사가 같은 카테고리에 실제 투자 중: "AI 기반 경영 인텔리전스 에이전트" 협약 [S11], AX Node·Node Studio [S02]

**Opportunity Sizing**:
- 수요 풀(후보 A+B 합산 논리): 정정 131사/년(5.1%) [S17] + 계속기업 주의 66곳(2.4%) [S20] — 상장사만 매년 **약 200개사**가 "행간을 못 읽어 사후 발견"의 직접 당사자 (계산식: 131+66, 중복 가능성 있어 (추정) 상한 표기).
- 카테고리 실증: 삼일 디지털 솔루션 매출 **200억 원**(2025-06 결산) [S07] — 이 계열 제품이 실제 매출 라인.
- 신뢰 격차: CEO 중 AI 성과 체감 12%(한국 14%) [S05·S14] — "근거를 제시하는(Trusted)" 에이전트가 겨냥하는 공백.

**플러그인 적합성**: 사람 ~2주(기업 1개사 결산 패키지 종합 리뷰 = 진단 엔게이지먼트 슬라이스) ✓ / Codex 플러그인 구조와 정합: 플러그인이 **가설 라이브러리(3종 가족)·교차 대조 절차·SOP 리포트 구조**를 제공하고 추론은 Codex가 수행 [S01, 02:35–02:51의 "방향성→추론" 구조 그대로] ✓ / 마감 내 구현: 코드보다 방법론(SKILL.md)+합성 데이터 설계 중심이라 가능 — 단 **가설군당 심은 문제 1~2개로 시나리오를 제한**해 깊이 희석을 방지(스코프 가드) ✓ / 본선에서 합성 데이터가 주어져도 가설 라이브러리를 그대로 재사용 ✓.

**과잉 설계가 아닌 이유**: 통합의 실체는 "가설 목록 하나 + 공통 리포트 파이프라인 하나"다. 범용 프레임워크가 아니라, 같은 결산 데이터 패키지 위에서 도는 **클러치 피처 하나**(행간 교차 대조 → 근거 인용 보고)이며, 폭은 가설 항목 수로만 늘어난다.

## 추천: 후보 C — 이유와 탈락 사유

**C 추천 이유**: ① 출제자가 과제를 단수의 에이전트("Trusted CEO Agent")로 명명하고 세 예시를 그 목표의 병렬 사례로 열거했다 [S01, 01:32–02:13] — 하나만 풀면 과제의 부분집합이고, 셋을 가설군으로 품으면 과제 그 자체다. ② 평가 축(기획·의도, 가설→추론→근거, SOP 설명 구조)을 가장 정면으로 시연한다 — 가설 라이브러리 설계가 곧 "기획"이다. ③ 본선(합성 데이터 + 심은 문제)에 무수정 재사용된다. ④ 구현 실체가 방법론+데이터 설계라 마감 내 가능하다.

**A 탈락**: 가장 안전하지만 "산재 데이터 총망라·행간"이라는 과제 본질 [01:36, 02:10] 이 단일 유형 오류 체커로 좁아진다 — 단, A의 탐지 시나리오는 C의 가설군 1로 그대로 흡수된다.
**B 탈락**: 근거 통계는 있으나 판단이 거시적이라 "근거를 자연어로 제시" [02:41–02:45] 데모의 선명도가 떨어진다 — B의 시나리오는 C의 가설군 2로 흡수된다.

## 추천안 → 문항 1·2 초안 뼈대

- **문항 1(한 문장)**: "여러 시스템에서 뽑은 결산 데이터 패키지(시산표·원장·전표·원가·현금흐름)를 받아, 회계 전문가의 가설 라이브러리를 따라 교차 대조 추론을 수행하고, 발견마다 근거 전표·계정을 인용해 경영진 보고 구조(SOP)로 설명하는 Codex 플러그인." — 쓰는 사람: 기업 CFO/재무팀·경영진(그리고 이들을 자문하는 회계법인 실무자)이 결산 마감·이사회 보고 전 "숨은 문제가 없는가"에서 막힐 때.
- **문항 2(왜 이 문제)**: 개별 숫자는 정상처럼 보여 시스템별 보고로는 행간의 문제가 안 보인다 [S01]. 그 결과 상장사 5.1%가 매년 재무제표를 사후 정정하고 [S17], 계속기업 '주의' 66곳이 강조사항으로야 드러난다 [S20]. 출제 기업이 직접 "산재 데이터에서 고민을 뽑아내는 에이전트"를 과제로 지목했고 [S01], 같은 카테고리에 실제 투자 중이다 [S11].
