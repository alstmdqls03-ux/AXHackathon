각 가설군은 `hypotheses/H1-accounting.md` `H2-cashflow.md` `H3-costing.md` **독립 파일**: [가설 정의 → 필요 데이터 → 교차 대조 절차 → 판단 기준(회계 기준 근거) → 근거 인용 규칙 → 리포트 섹션 템플릿]. 축소 = 해당 파일과 데이터 컬럼 삭제로 끝 (DECISION 조건 ②).

구조(회계 실무 검토보고 관행 반영, S01 [03:49–04:06] "SOP에 맞닿은 설명 구조"):

### 공통 실패·정보부족 동작 (문항 3 대응 — 모든 후보 공통)

1. **필수 파일 누락** → 가설군별 요구 데이터 매트릭스로 판정, 없는 가설군만 "검토 불가(사유: monthly_series.csv 부재)"로 보고서에 명시하고 나머지 가설군은 정상 수행 (모듈 독립성).

## 후보 1 — 순수 SKILL.md 방법론 플러그인

**구현 형태**: `.codex-plugin/plugin.json` + `SKILL.md` + `hypotheses/*.md` 3개 + 데모 데이터. 실행 코드 없음 — Codex가 SKILL.md의 절차를 따라 스스로 python/pandas 코드를 즉석 작성해 집계·교차 대조하고 리포트를 쓴다.

**데모 시나리오**:
- 입력: `결산 패키지를 행간 감사해줘. 데이터: data/demo/` (또는 스킬 자동 트리거)
- 동작: SKILL.md가 지시 → ① 데이터 인벤토리·스키마 확인 ② H1~H3 각 모듈의 교차 대조 절차 수행(Codex가 즉석 코드로 계산) ③ 이상 후보를 판단 기준에 대조 ④ 근거 좌표 인용해 report.md 작성
- 출력 예시(발견 1건 발췌):

```markdown
### F-01 [H1·회계처리] 수선비 8.4억 자본화 — 이익 과대 의심 (심각도 高 · 확신도 中)
- 근거: 전표 #4712 (2025-12-28) 차) 유형자산 840,000,000 / 대) 미지급금 — 적요 "3호기 대수선", 작성자=승인자 동일
- 행간: 수선비 계정 전년比 -78% ↔ 유형자산 +12% 동시 발생, 자본화 증빙 필드 공란
- 판단 기준: K-IFRS 1016 후속원가 — 내용연수 연장 등 요건 미충족 시 비용 처리
- 영향: 영업이익 8.4억 과대 (영업이익률 7.2%→5.1%)
- 권고: 공사 내역서 확인 후 수정분개 차) 수선비 / 대) 유형자산 840,000,000
```

**비용·리스크**: 구축 비용 최소(문서+데이터만) — 마감 내 확실. 리스크: 수치 계산을 매 실행 Codex 재량에 맡겨 **재현성·정확성 편차** (같은 데이터에 실행마다 다른 집계 경로). 심사자 2회 실행 시 결과 흔들리면 신뢰(Trusted) 콘셉트에 자해.
**실패 동작(추가)**: 계산 불일치 감지 시(차대 불일치 등 자가 검산) 해당 발견의 확신도를 낮추고 재계산 — 그러나 강제 장치 없음.

## 후보 2 — SKILL.md + 결정론 체커 하이브리드 ★추천

**구현 형태**: 후보 1 + `tools/checks.py` 1개(표준 라이브러리만, 의존성 0). 스크립트는 가설군별 **결정론 교차 대조 계산**만 수행해 `findings_candidates.json`(이상 후보 + 원본 좌표)을 산출: H1) 차대 검산·기말 전표 스캔·계정 급변 탐지 H2) 이익-CF 괴리·회전일수·거래처 편중 H3) 배부 기준 대안 재계산(노동↔기계시간). **판단·해석·보고는 Codex**: 후보를 기각/확정하고(오탐 가드), 왜 문제인지 행간을 설명하고, 기준을 인용해 SOP 리포트를 쓴다.

**데모 시나리오**:
- 입력: 동일 (`data/demo/` 패키지 + 스킬 호출 한 줄)
- 동작: ① `python3 tools/checks.py data/demo/` → 이상 후보 JSON (예: `{"id":"H3-01","type":"allocation_sensitivity","product":"B","margin_labor_basis":0.12,"margin_machine_basis":-0.03,"rows":["cost_allocation.csv#L14-L18"]}`) ② Codex가 각 후보를 원본 데이터로 검증·기각·병합 ③ 확정 발견을 판단 기준·영향·권고와 함께 서술 ④ report.md (+ 부록: 검토한 가설 전체와 통과 항목)
- 출력 예시(H3 발췌 — 계산은 스크립트, 해석은 Codex):

```markdown
### F-03 [H3·원가배분] 제품 B는 회계상 흑자, 경제적 실질은 손실 (심각도 高 · 확신도 高)
- 계산 근거(재현 가능): 간접비 42억을 현행 노동시간 기준→B 배부 6.3억(마진 +12%);
  기계시간 기준 재배부→B 배부 18.9억(마진 -3%) [checks.py H3, cost_allocation.csv 14–18행]
- 행간: B는 자동화 라인(기계 7,200h vs 노동 900h) — 노동시간 배부가 B 원가를 A로 전가
- 판단 기준: 배부 기준은 자원 소비 인과관계 반영(관리회계 원칙) — 기계집약 제품에 노동시간 기준은 인과 왜곡
- 영향: 제품별 의사결정 왜곡 — B 증산·A 축소 전략은 실질 손실 확대
- 권고: 활동기준(기계시간·셋업횟수) 재배부 후 제품 포트폴리오 재평가
```

**비용·리스크**: 스크립트 ~200줄(표준 lib) + 문서 — 마감 내 가능(오늘 밤 체커+데이터, 내일 오전 검증 = DECISION 일정 게이트와 정합). 리스크: 체커 버그 → **완화: 후보 1로 자동 강등** — SKILL.md에 "checks.py 실행 실패 시 동일 절차를 직접 계산으로 수행"을 명시(가설 정의가 선언적으로 hypotheses/*.md에 있으므로 가능). 심사 환경은 python3만 요구(외부 패키지 0).
**실패 동작(추가)**: ① checks.py 실행 불가 → 후보 1 모드로 자동 폴백(리포트에 "수동 계산 모드" 명시) ② 후보 JSON과 Codex 재검산 불일치 → 해당 발견 확신도 강등 + 불일치 내역 보고.

**왜 이 분업인가**: 계산의 재현성(결정론)과 행간 해석(LLM 추론)을 분리 — 출제자의 "AI한테 **추론의 방향성**을 주면 AI가 추론을 수행하고 근거를 제시" [S01, 02:35–02:51] 구조를 그대로 구현. "AI가 아니어도 되는 것"(집계·검산)은 스크립트에, **AI라서 되는 것**(오탐 기각, 인과 서사, 기준 적용, 권고)은 Codex에 [S01, 02:23–02:33 감점 기준 회피].

### 확정 피치 — 후보 2 "행간 감사 에이전트" (가칭, 하이브리드)

**클러치 피처 한 문장**: "결산 데이터 패키지(시산표·전표·월별 시계열·채권 연령·원가 배부표)를 넣으면, 회계 전문가의 가설 라이브러리 3종(회계처리 오류·현금흐름 이상·원가배분 왜곡)을 따라 결정론 교차 검산과 Codex 추론으로 '행간에 숨은 문제'를 찾아, 발견마다 전표·행 단위 근거를 인용한 경영진 SOP 보고서를 만들어 주는 Codex 플러그인."

**왜 이 방식이 이 문제를 가장 잘 푸는가**:
1. 과제 원문 "산재된 데이터를 총망라해 기업의 고민을 뽑아내는 에이전트" [S01, 01:36–01:44] — 가설 라이브러리가 '총망라'를, 교차 검산이 '행간' [02:10] 을 구현한다.
2. 평가자가 정의한 AI의 가치 — "추론의 방향성을 주면 추론을 수행하고 **근거를 자연어로 제시**" [S01, 02:35–02:51] — 가 아키텍처 그 자체다: hypotheses/*.md(방향성) → Codex(추론) → 인용 규칙(근거).
3. "단순 정리는 AI 아니어도 가능" 감점 [S01, 02:23–02:33] 회피 — 집계는 스크립트가 하고, Codex는 오탐 기각·인과 서사·기준 적용이라는 AI 고유 몫만 한다.
4. "신뢰성 있는 설명 + 조직의 일하던 방식(SOP)" [S01, 03:49–04:06] — 리포트가 회계 검토보고 관행(근거→기준→영향→권고→확신도)을 따르고, 모든 수치는 재현 가능(결정론 계산 + 좌표 인용).
5. "문제가 완전히 디파인되지 않은" 입력 [S01, 02:51–02:57] — 가설 구동 수색 + 발견 0건 정직 보고 + 검토 불가 항목 명시.

**데모 계획** (심사자 관점, 비주얼 포함):
1. 설치: 로컬 marketplace 등록 → 플러그인 설치 (README 3줄).
