## 핵심 힌트 ① 출제 과제: Trusted CEO Agent (01:32–02:13)

> "이번에 저희가 출제한 **트러스티드 CEO 에이전트**의 목표는 그겁니다. 기업에 산재되어 있는 굉장히 많은 데이터들을 총망라해서 **기업의 고민을 뽑아내는 에이전트**."

- 다양한 시스템에 들어가 있는 **가상의 합성 데이터를 많이 제공**할 것 (02:46–03:04: 데이터 안에 **의도적으로 심어 놓은 문제점들**이 있음)
- 종합적 횡관(橫觀)을 읽어 뽑아낼 수 있는 것의 예시 (01:59–02:13):
  1. **틀린 회계 처리를 찾는다**
  2. **현금 흐름상의 문제점을 찾는다**
  3. **원가 배분이 잘 안 되어 특정 제품이 손실인데 겉으로 안 드러나는 것을 찾는다**
- 즉 "**데이터의 행간 속에 묻어 있는 요소를 찾는 것**"이 포인트

## 핵심 힌트 ② 평가 기준 (02:16–03:07)

1. **어떻게 만들었는가보다 어떤 의도로 기획했는가**가 더 중요
2. 자료를 잘 정리해 의사결정자에게 보여 주는 것만으로는 부족 — "AI가 아니어도 할 수 있다"
3. AI의 진짜 가치 = **추론의 방향성을 주면 AI가 추론을 수행하고, 그 근거를 자연어로 제시**해서 사람이 고민의 출발점으로 삼게 하는 것 (근거 제시·설명가능성)
4. 기업 고객의 문제는 **완전히 디파인되지 않은 채** 주어질 수 있음 — 문제 정의 능력 자체가 평가 대상
5. (Ch.5 연계) 신뢰성 있는 설명 + 조직의 기존 업무 방식·SOP에 맞닿은 설명 구조 = UX

## 플러그인 설계에 대한 시사점 (해석)

- 방향: **재무·회계 데이터 더미에서 숨은 이상(잘못된 회계처리 / 현금흐름 이상 / 원가배분 왜곡)을 찾아내고, 근거를 자연어로 설명하는 에이전트** — 영상이 예시로 준 3가지가 사실상 스펙.
- 차별화 포인트는 탐지 자체보다: ① 추론 방향성(가설) 설정 → ② 근거 데이터 인용 → ③ 회계 실무 SOP에 맞는 설명 구조로 전달.
- "단순 대시보드/요약"은 명시적으로 감점 사유로 언급됨.
