# step-02 — 공개 근거 대장 구축

`../orchestration/prompts/RULES.md`의 공통 규칙을 적용하라. 이 스텝만 수행하고 정지한다.

## 목적

지금까지의 모든 근거를 단일 대장(ledger)으로 정리해, 이후 문서(문제·솔루션·문항 답변)가 대장 번호로 인용할 수 있게 한다.

## 작업

1. `context/sources.md`를 아래 대장 형식으로 재구축한다 (기존 내용은 흡수·재정리):

   | # | 주장/사실 | 출처 URL | 수집일 | 유형(사실/추정) | 비고 |
   |---|---|---|---|---|---|

2. `intent.md` · `company.md` · (있다면 기존 problem.md)의 모든 주장·숫자가 대장에 등재됐는지 대조하고 누락분을 추가한다.
3. 각 URL이 로그인 없이 열리는지 확인한다. 접근 불가 URL은 대체 공개 출처를 확보한다.
4. 근거를 못 찾은 주장은 삭제하거나 (추정)으로 강등하고 그 사실을 기록한다.

## 산출물

- `context/sources.md` (근거 대장)

## 게이트

- [ ] intent/company의 모든 숫자·주장이 대장 번호로 추적 가능
- [ ] 전체 URL 접근성 확인 완료 (확인 일시 기재)
- [ ] 출처 없는 숫자 0개

## 종료

`context/JOURNAL.md`에 규격 보고 후 정지.
