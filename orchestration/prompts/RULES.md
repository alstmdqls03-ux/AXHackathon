# RULES.md — 모든 스텝 공통 규칙 (스텝 수행 전 필독)

## 진행 방식

1. **한 번에 한 스텝만.** 지시받은 step-NN만 수행하고, 다음 스텝을 미리 진행하지 않는다.
2. 스텝을 마치면 `context/JOURNAL.md`에 아래 규격의 보고를 **추가(append)** 하고 **정지**한다. 오케스트레이터의 다음 지시를 기다린다.
3. **step-03(문제 선정)과 step-04(솔루션 확정) 완료 후에는 강제 정지** — 사용자 승인 없이는 어떤 후속 작업도 시작하지 않는다.

## JOURNAL.md 보고 규격 (스텝당 1개 섹션, 항목 4개 고정)

```markdown
## [step-NN] <스텝 이름> — <YYYY-MM-DD HH:MM>
### 한 일
### 핵심 발견과 결정
### 게이트 체크
(스텝 파일의 게이트 항목별 ✅/❌ + 한 줄 근거)
### 오케스트레이터에게 요청
(없으면 "없음")
```

## 근거 규칙 (모든 산출물 공통)

- 모든 주장·숫자에 **출처 URL + 수집일(YYYY-MM-DD)** 을 붙인다.
- **사실과 추정을 구분**하고, 추정은 반드시 `(추정)` 표기 + 추정 근거를 쓴다.
- **출처 없는 숫자 절대 금지.** 비공개 내부 정보·확인 불가한 사적 경험 불인정.
- URL은 로그인 없이 접근 가능해야 한다 (AI 심사자가 직접 검증).

## 가중치

`context/hint-video-transcript.md` · `hint-video.md` · `intent.md` (**출제 기업 직접 힌트**) **> 공식 자료 > 제3자 공개 자료 > 유저 목소리.**
해석·우선순위가 충돌하면 힌트 영상이 우선한다. 문제·솔루션 문서는 힌트 영상 근거(타임스탬프)를 명시적으로 인용한다.

## 불변 규칙

- 이 대화는 **원본 그대로 제출 로그**가 된다: API 키·토큰·비밀번호·개인정보 입력 금지.
- `logs/` 아래 파일 절대 편집·발췌·삭제 금지 (실격 사유). **세션 로그 형태의 파일(`*.jsonl` 등)은 위치와 무관하게 동일하게 불가침이다** (예: context/ 아래에 로그 사본이 있어도 수정·삭제 금지). 그 외 context/ 리서치 산출물(md 문서)은 편집 가능.
- 다른 참가 회사 언급·참조 금지. 산출물은 전부 이 회사 폴더 안에만 생성.
- 스코프: 사람이 ~2주 걸릴 업무를 대체하는 **클러치 피처 하나**. 범용 프레임워크·과잉 설계 금지.
