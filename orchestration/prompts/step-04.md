# step-04 — 솔루션 후보 + 확정 피치 ⛔ 승인 게이트

`../orchestration/prompts/RULES.md`의 공통 규칙을 적용하라. 이 스텝만 수행하고 정지한다.
**승인된 문제(step-03에서 사용자가 선정)에 대해서만 수행한다. ⛔ 완료 후 사용자 승인 전까지 어떤 후속 작업도 시작 금지.**

## 작업

솔루션 후보 2~3개를 만든다. 후보마다:

1. **데모 시나리오**: 실제 입력 예시 → 플러그인 동작(절차·판단 기준) → 출력 예시
2. **구현 형태**: SKILL.md 중심인지, MCP 서버(.mcp.json)·실행 코드가 추가로 필요한지
3. **구현 비용과 리스크**: 마감 내 가능성 포함
4. **정보 부족·실패 상황에서의 동작** (질문 문항 3 요구사항)

그다음 1개를 추천하는 **확정 피치**를 쓴다:
- 클러치 피처 한 문장
- 왜 이 문제를 이 방식이 가장 잘 푸는가 — 힌트 영상 근거(타임스탬프) 인용
- 데모 계획 (데모 가능 + 비주얼 요소)

## 산출물

## 게이트

- [ ] 데모 시나리오에 실제 입력·출력 예시 포함
- [ ] 실패·정보부족 동작 포함
- [ ] 피치가 질문 문항 1·3 초안으로 옮겨질 수준
