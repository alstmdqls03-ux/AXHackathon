# step-01 — 회사 컨텍스트

`../orchestration/prompts/RULES.md`의 공통 규칙을 적용하라. 이 스텝만 수행하고 정지한다.

## 목적

공개 자료로 이 회사의 사업·산업·고객 컨텍스트를 파악해, 문제 후보 도출의 배경을 만든다.

## 입력

- `context/intent.md` (최상위 가중치 — 조사의 방향타)
- `../RESEARCH_PLAYBOOK.md` 출처 우선순위·기록 규칙

## 작업

1. 사업 모델 · 주요 제품/서비스 · 고객 구성 (공식 자료 우선)
2. 최근 1~2년 주요 행보 · 발표 · 공시 · 보도
3. 산업 동향과 경쟁 환경 (공개 리포트 · 기사)
4. 유저/고객의 공개 피드백 (리뷰 · 커뮤니티 · 포럼)
5. intent.md의 문제의식과 연결되는 관찰에 `[의도연결]` 태그를 붙여 명시

이전 지시로 이미 조사한 내용이 있으면 이 구조로 흡수·재정리한다 (리서치 산출물 편집은 허용 — logs/와 무관).

## 산출물

- `context/company.md` — 모든 주장에 URL + 수집일, 추정은 (추정) 표기

## 게이트

- [ ] 주장마다 출처 URL + 수집일
- [ ] `[의도연결]` 관찰 3개 이상
- [ ] (추정) 표기 준수, 출처 없는 숫자 0개

## 종료

`context/JOURNAL.md`에 규격 보고 후 정지.
