# step-03 — 문제 후보 비교 + Opportunity Sizing + 선정 제안 ⛔ 승인 게이트

`../orchestration/prompts/RULES.md`의 공통 규칙을 적용하라. 이 스텝만 수행하고 정지한다.
**⛔ 이 스텝 완료 후에는 사용자 승인 전까지 어떤 후속 작업도 시작 금지.**

## 목적

힌트 영상(최상위 가중치)과 근거 대장에 기반해 문제 후보를 비교하고 1개를 추천한다. **최종 선정은 사용자가 한다.**

## 작업

문제 후보 3개를 도출한다. 후보마다:

1. **문제 정의**: 누가 / 어떤 상황에서 / 어디서·왜 막히는가
2. **근거**: `intent.md` 타임스탬프 최소 1개 + 대장 번호(`sources.md #N`) — 출제자 힌트 근거가 가장 중요하며 이를 명시
3. **Opportunity Sizing**: 공개 숫자 기반 대략치 — 계산식과 출처 제시, 추정은 (추정) 표기
4. **플러그인 적합성**: 사람 ~2주 분량 업무인가 / Codex 플러그인·SKILL.md로 데모 가능한가 / 마감 내 구현 가능한가

그다음 **비교표**를 만들고 **1개를 추천**한다 — 추천 이유와 나머지 탈락 이유 포함.

## 산출물

- `context/problem.md` (기존 파일이 있으면 이 구조로 재작성 — 기존 발견은 근거로 흡수)

## 게이트

- [ ] 각 후보에 힌트 영상 근거 ≥1 (타임스탬프 인용)
- [ ] sizing에 계산식 + 출처, 추정 표기
- [ ] 추천안이 질문 문항 1·2 초안으로 바로 옮겨질 수준

## 종료

`context/JOURNAL.md`에 규격 보고 — "오케스트레이터에게 요청" 란에 **"문제 선정 승인 대기"** 를 명시하고 완전 정지.
