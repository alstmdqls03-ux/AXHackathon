# trade-decision-report — 매매 의사결정 설득 리포트

## 목적

초보 투자자의 두 가지 질문 — "나 손해 본 거야?"와 "지금 어떻게 해야 해?" — 에 대해, **정답이 아니라 납득 과정**을 제공한다. 리포트는 [① 진단 → ② 선택지 병렬 비교 → ③ 실행 체크리스트]의 3단 고정 구조이며, 순위·추천을 제시하지 않고 결정을 사용자에게 남긴다.

## 절대 규칙 (코드가 강제하지만, 너도 지켜라)

1. **리포트 본문을 직접 작성하지 마라.** 최종 리포트는 반드시 `scripts/render.py`가 생성한 `report.md` 파일이다(마지막 줄에 report-id 스탬프). 사용자에게는 그 파일 내용을 그대로 보여줘라.
2. **숫자를 만들어내지 마라.** 모든 수치는 `scripts/diagnose.py`의 산출물이다. 네가 작성하는 선택지 JSON의 서술에는 숫자 리터럴을 쓸 수 없고, `{metrics.<id>}` 플레이스홀더만 허용된다.
3. **추천하지 마라.** "가장 좋다", "추천한다", "오를 것이다" 류의 표현은 render.py가 거부한다. 각 선택지는 [일어나는 일 / 근거 / 확정되는 것 / 알 수 없는 것]을 담담하게 서술한다. "알 수 없는 것"에는 이 리포트가 미래를 모른다는 사실을 정직하게 쓴다.
4. **정보가 없으면 물어라.** 필수 입력이 없을 때 임의 추정값을 넣지 마라 — diagnose.py가 어떤 값이 왜 필요한지 알려주면 그대로 사용자에게 되물어라.

## 절차

1. **입력 수집**: 사용자로부터 (a) 거래 내역 CSV — 헤더 `date,ticker,side,qty,price_usd,fee_usd,fx_krw_per_usd`, (b) 현재가(USD), (c) 현재 환율(KRW/USD)을 받는다. CSV가 없으면 대화로 받아 CSV로 만들어 준다.
2. **진단 (결정론 계산)**:
   ```bash
   python3 scripts/diagnose.py trades.csv --price <현재가> --fx <환율> --out diagnosis.json
   ```
3. **선택지 JSON 작성 (너의 유일한 작문 구간)**: `diagnosis.json`의 metrics를 보고, 4개 선택지(hold/partial_sell/dca/sell_all — 이 id 그대로, 각 1개)를 작성한다. 형식은 `samples/options.sample.json` 참조. 각 필드 ≤200자, 근거(`evidence`)는 실제 존재하는 `metrics.<id>` 참조만, 서술 속 숫자는 `{metrics.<id>}` 플레이스홀더만.
4. **검증·조립**:
   ```bash
   python3 scripts/render.py diagnosis.json options.json --out report.md
   ```
   거부되면(아래 표) 위반 사항을 고쳐 3번부터 다시 — 거부 사유를 사용자에게 숨기지 마라.
5. **전달**: `report.md` 내용을 그대로 보여주고, 사용자가 선택지를 고르면 `--chosen <id>`로 다시 렌더해 해당 체크리스트만 담은 최종본을 준다.

## 실패·정보 부족 시 동작

| 상황 | 코드 | 행동 |
|---|---|---|
| 현재가/환율 미제공 | `REQUIRED_INPUT_MISSING` | 어떤 값이 왜 필요한지 그대로 전달하고 되묻는다. **임의 값 대입 금지.** |
| CSV 형식·값 오류 | `CSV_INVALID` | 행 번호별 오류 목록을 보여주고 수정을 요청한다. |
| SELL 내역·복수 종목·주식 외 자산 | `UNSUPPORTED_SCOPE` | 데모 범위(단일 종목, 미국 주식, BUY 내역)를 설명하고 범위 내 입력을 안내한다. |
| 근거가 진단에 없는 id | `EVIDENCE_NOT_GROUNDED` | 선택지 JSON을 고쳐 재시도 (실존 metric만 인용). |
| 서술에 숫자 리터럴 | `UNGROUNDED_NUMBER` | 숫자를 `{metrics.<id>}` 플레이스홀더로 바꿔 재시도. |
| 추천·비교·예측 어휘 | `RANKING_LANGUAGE_DETECTED` | 해당 표현을 중립 서술로 바꿔 재시도. |
| 사용자 의도가 모호 ("주식 어떡해?") | — | 종목·보유 여부·거래 내역 유무를 묻는 질문 1개로 구체화한 뒤 진행. |
| 사용자가 "그래서 뭘 사/팔까?"라고 추천을 요구 | — | 리포트는 순위를 제시하지 않으며 결정은 사용자의 몫임을 설명하고, 판단 재료(진단·비교)를 다시 안내한다. |

## 판단 기준 (선택지 서술 작성 시)

- **일어나는 일**: 그 행동 직후 계좌에서 실제로 일어나는 일만. 미래 서술 금지.
- **근거**: 그 서술을 뒷받침하는 진단 수치의 id. 렌더러가 `id = 값` 형태로 병기해 준다.
- **확정되는 것**: 그 선택으로 되돌릴 수 없게 되는 것 (손익 확정, 원금 추가 노출 등).
- **알 수 없는 것**: 이 리포트가 계산할 수 없는 것 — 반드시 정직하게. 이 필드가 신뢰(납득·안심)의 핵심이다.

## 범위·한계 (사용자에게도 밝혀라)

- 데모 범위: 단일 종목, 미국 주식, BUY 거래 내역. 세금은 단순화 계산(연 250만 원 기본공제 초과분의 22%, 손익통산·환율 차익 미반영).
- 이 스킬은 투자 자문이 아니며 시세·뉴스를 조회하지 않는다. 모든 수치는 사용자 입력에서 결정론적으로 유도된다.
