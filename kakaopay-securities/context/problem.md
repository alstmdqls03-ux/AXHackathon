# problem.md — 문제 후보 비교 + Opportunity Sizing + 선정 제안

> **가중치 규칙**: 출제자 힌트 영상(S1, `intent.md`)은 출제 기업 직접 힌트로 리서치 전체에서 최상위 가중치. 문제 선정·해석이 다른 출처와 충돌하면 S1이 우선한다. 인용은 근거 대장(`sources.md`)의 Sx-n/I-n 번호.
>
> 검증 방식: 후보 3개를 서로 다른 렌즈의 판정단 3인(출제자 정합성 / 스코프·실행 가능성 / 차별화·근거 강도)이 독립 채점 + 적대적 반박 — 합산 **C 13.5 > A 11 > B 5**. 반박 내용은 추천안의 설계 제약으로 흡수(§5).

## 0. 공통 전제 (출제자가 확정한 것)

- 문제 도메인: **"투자자들의 매수 매도를 도와주는 것"** — S1-1 [01:03]. 초보 투자자의 두 니즈: ① "내가 손해 보지 않았다라든지"(S1-2 [01:08]) ② "매수 매도를 어떻게 해야 될지 잘 모르겠는데 어떻게 도움을 받을 수 있을까"(S1-3 [01:12]). "많은 분들이 괴로워한다라는 생각이 들어서 일단 이 문제를 냈습니다"(S1-4 [01:19]).
- 평가 기준: 수학적 최적점 아님(S1-7 [01:32]) / "납득해서 안심하고 그다음을 실행할 수 있게끔 정보를 충분히 주는지"가 **제일 중요**(S1-8 [01:41]) / "설득해 나가는 과정"·"과정 설계"를 본다(S1-5 [01:26], S1-9 [01:51]).
- 형태 힌트: 형상 없는 에이전트(S1-15 [03:29]~[04:02]), 출제자 본인 Codex 사용(S1-14 [03:11]).

## 1. 후보 A — 매매 복기·손익 설득 리포트 ("나 손해 봤어?")

- **문제 정의**: 카카오페이증권형 초보 투자자(2040 비중 70%대 이상, S6-1)가 / 매도 직후 또는 보유 중 평가손실 상태에서 / 수수료·세금·환율·평균단가까지 반영한 **실질 손익과 그 이유**를 설명받지 못해 막힌다. 하루 수십 번 앱만 확인하고(S27-1, I-5) "물타기? 손절?" 사이에서 엇갈린 조언에 방치된다(S28-1).
- **근거**: 힌트 S1-2 [01:08]·S1-4 [01:19]. 실증: 처분효과 — 이익 포지션 매도확률이 손실의 2배(41% vs 22%, S2-2), 신규투자자 60% 손실·비용 차감 후 -1.2%(S3-1), 수익 중에도 불안(S30-1).
- **Opportunity Sizing**: (추정) 월 거래자 50만 명(S6-1) × 신규투자자 손실 경험 비율 60%(S3-1) ≈ **월 ~30만 명**이 '손해 봤나' 판단 상황에 노출. 시장 전체는 개인투자자 1,410만 명(S24-1).
- **플러그인 적합성**: 결정론적 계산 코어(체결 내역 → 실질 손익 분해) + 설명 계층 = 애널리스트/CX ~2주 분량. 샘플 거래 데이터로 마감 내 데모·정상/예외 로그 검증이 **가장 확실**. 단, "그다음 실행"(S1-8)이 구조적으로 없고 계산 정밀도 경쟁으로 흐르면 출제자가 부정한 프레임(S1-7)에 근접.

## 2. 후보 B — 주문 실행 내비게이터 ("어떻게 사고팔아?")

- **문제 정의**: 초보 투자자가 / 매수·매도 주문을 내는 순간(주문 유형·수량·시간대·환전, 특히 해외주식) / 실행 방법을 몰라 막힌다.
- **근거**: 힌트 S1-3 [01:12]. 회사 공식 인정: "언제 사고 팔아야 할지 판단하기 어려워"(S13-1), 공식 콘텐츠의 문제 인정(S9-1), 직접투자 성과 열위(S26-1).
- **Opportunity Sizing**: (추정) 미국 주식 보관액 300조 원(S25-1) 시장의 진입 장벽 + 주식 모으기 160만 명(S4-1)이 수동 매매로 전환할 때 전원 노출.
- **플러그인 적합성**: 절차 안내는 정적 지식 위주 — ~2주 업무보다 FAQ 챗봇에 가깝고(과소 스코프), 기구축 상담 시스템(S1-11)의 재발명으로 읽힐 위험. 구체 실행 단계는 특정 앱 UI를 전제하게 되는데 출제자는 형상을 부정(S1-15). 결정적으로 S13-1·S9-1의 원문은 "**언제** 사고팔지"(판단)의 고통이지 "**어떻게** 주문하는지"(조작)가 아니며, S29-1("직관적·간편")은 조작 마찰이 작다는 반대 증거.

## 3. 후보 C — 매매 의사결정 설득 에이전트 ("지금 어떻게 해?") ★ 추천

- **문제 정의**: 초보 투자자가 / "이 종목 어떡하지?"의 순간(보유 중 손실·수익, 매도 고민)에 / 정보는 많지만(S1-6) **자기 상황에 적용해 납득시켜 주는 설명이 없어 다음 행동을 실행하지 못한다**(S1-8). 유저 원문이 이 결합을 그대로 보여줌: "-20%인데 물타기 알려주세요" → 존버/분할/손절로 엇갈린 답변 속 방치(S28-1). 회사 대응은 임계값 자동화(S12-1)에 머물러 "왜"가 빈칸(I-1), 업계도 요약·번역 단계(I-2, 추정).
- **근거**: 힌트 S1-1~S1-4(두 고통 모두) + 평가 기준 S1-5/8/9 + 에이전트 비전 S1-15. 실증은 A·B의 근거를 전부 흡수(S2-2, S3-1, S13-1, S26-1, S27-1, S28-1, S30-1).
- **Opportunity Sizing**: (추정) 월 거래자 50만 명(S6-1)의 모든 매매 판단 순간이 대상 — 손실 경험자만 잡아도 월 ~30만 명(S3-1 60% 적용). 회사 관점 업무 번역: 상담사가 건별 응대하는 "이 거래 손해인가요/어떻게 하나요" 문의 대응 + CX·리서치의 설명 콘텐츠 설계 ≈ 사람 ~2주 분량.
- **플러그인 적합성**: 산출물을 **"의사결정 리포트 하나의 흐름"**으로 고정 — [① 진단(실질 손익 분해 계산) → ② 선택지 비교(홀드/분할/손절/부분매도 — 서열화 없이 각각의 근거·리스크) → ③ 사용자가 고른 행동의 실행 체크리스트]. A의 결정론적 계산 코어를 그대로 재사용하고 ②를 ①의 계산 산출물에만 접지, ③을 고정 템플릿으로 제한하면 A 대비 추가 구현 비용은 제한적(판정단 렌즈 2 분석). Codex 플러그인+SKILL.md 데모: 샘플 보유/거래 데이터 → 리포트 생성 → 정상/예외 로그.

## 5. 추천: 후보 C (A의 계산 코어를 기반으로, 아래 제약을 걸어)

**추천 이유**: ① 출제자가 유일하게 "제일 중요"라 못 박은 기준(S1-8)의 '납득→안심→그다음 실행' 사슬을 산출물 구조로 그대로 옮긴 유일한 후보. ② 두 고통의 병렬 지목([01:08][01:12])과 유저 원문의 결합 증거(S28-1)가 "진단과 다음 행동은 한 질문"이라는 해석을 지지. ③ 회사 로드맵 공백(I-1)·업계 공백(I-2)을 정조준하고 에이전트 비전(S1-15)·2026 전략 '개인화+에이전트'(S15-2)와 정합. ④ 문항 1·2 전환력 최고 — 문항 2에 출제 발언 3개([01:08][01:12][01:19])를 모두 인용 가능.

**탈락 이유**: A — 근거는 최강이지만 출제 도메인의 절반(진단)에서 멈춰 [01:41]의 '그다음 실행'이 구조적으로 없고, '손익 계산기'로 읽히면 출제자가 2회 부정한 수학 프레임([01:32][01:51])과 충돌. B — 고통 ②의 원문 문맥([01:12]→[01:29]→[01:32])상 조작법이 아닌 판단의 막막함인데 이를 how-to로 오독한 후보. 기구축 상담 시스템(S1-11) 재발명 + 형상 부정(S1-15)과 충돌 + 가장 뻔한 답.

**판정단 적대 반박을 흡수한 설계 제약** (step-04에서 솔루션 스펙으로 구체화):
1. **보조수단성 준수** (반박: "맞춤 매매 조언 봇으로 재분류될 위험"): 선택지를 서열화·추천하지 않는다. 각 선택지의 근거·리스크·비용을 병렬 제시하고 결정은 사용자가 — 금융위 7대 원칙 '보조수단성'(S10-1)과 미래에셋 "마지막 결정은 사람"(S23-1) 프레임에 명시적으로 올라탄다.
2. **스코프 고정** (반박: "리포트 한 흐름 약속이 설계로 강제되지 않으면 비대해짐"): ② 선택지 비교의 근거는 ① 진단의 계산 산출물(실질 손익 분해 수치)에만 접지 — 시장 데이터·뉴스 예측으로 확장 금지. ③ 실행 단계는 고정 템플릿 체크리스트. 자산군은 데모 범위로 한정.
3. **'수학적 최적점' 오독 차단** (반박: "계산 정밀도 경쟁"): 계산은 설득의 재료이지 목적이 아님을 README·question.md에 명시 — "정답이 아니라 납득 과정"(S1-9) 프레임 유지.
4. **업계 공백 주장은 (추정)으로만** (반박: "토스 AI 시그널 반례 가능"): "공개 자료상 미확인(I-2, 추정)"으로 표현하고, 차별화의 본체는 공백 주장이 아니라 [01:41] 사슬의 구조적 구현에 둔다.

## 6. 문항 초안 뼈대 (승인 후 확정)

- **문항 1(무엇을)**: 초보 투자자의 보유/거래 내역과 "이 종목 어떡하지?"라는 의도를 받아, [실질 손익 진단 → 선택지별 근거·리스크 비교 → 선택한 행동의 실행 체크리스트]를 설득 구조로 담은 의사결정 리포트를 생성하는 Codex 플러그인 — 결정은 항상 사용자에게 남긴다(보조수단성).
- **문항 2(왜)**: 출제자가 직접 지목한 두 고통([01:08][01:12], "이 문제를 냈습니다" [01:19]) + 정량 실증(신규투자자 60% 손실 S3-1, 처분효과 2배 S2-2) + 유저 원문(S28-1) + 회사 로드맵 공백(I-1, 추정).
