# RESEARCH_PLAYBOOK.md — 회사별 리서치 방법

> **리서치는 플러그인을 위한 수단이다.** 모든 발견은 플러그인 설계 결정 하나로 연결되어야 하며, 연결되지 않는 조사는 중단한다.
> 반드시 해당 회사 폴더 세션에서 수행한다. 산출물: `<회사>/research/problem.md`, `<회사>/research/sources.md`

## 출처 우선순위 (근거는 전부 공개 자료 + AI가 접근 가능해야 인정)

1. **출제자 컨텍스트 — 기업 인터뷰 영상** (조코딩 · 조코딩AX파트너스 채널)
   - https://www.youtube.com/watch?v=aBuoojGjyf4
   - https://www.youtube.com/watch?v=5iRf37Z8Wd4
   - https://www.youtube.com/watch?v=y0SsTKDt8tQ
   - ⚠️ 어느 영상이 어느 회사인지 먼저 매핑하고, **회사 폴더 안에서는 그 회사 영상만 참조** (회사 간 분리 원칙).
   - 영상에서 언급되는 문제는 본선 출제 문제 — 예선에 활용해도 무방(FAQ Q8). 단 본선에서 예선·본선 답변을 함께 평가함을 감안.
2. **공식 자료**: 회사 홈페이지·공식 블로그·IR·보도자료·공식 문서·공시
3. **제3자 공개 자료**: 기사, 공개 산업 리포트, 공개 통계
4. **유저 목소리**: 앱스토어 리뷰, 공개 커뮤니티·포럼의 피드백

## 불인정 근거 (규정 — 사용 금지)

기업 내부 비공개 정보 / 확인할 수 없는 사적 경험 / 출처 없는 숫자

## 기록 규칙

- 근거로 쓸 사실을 발견한 **즉시** `sources.md`에 `URL + 한 줄 요약`으로 기록한다 (나중에 다시 못 찾는다).
- 숫자를 인용하면 반드시 URL을 동반한다.
- URL은 **로그인 없이 열리는지 확인**한다 — AI 심사자가 직접 접근해 검증한다.

## 산출물 템플릿 — `research/problem.md`

1. **컨텍스트**: 회사·산업·고객 현황 (3~5줄, 출처 링크)
2. **문제 후보** (2~3개): 각각 "누가 / 어떤 상황에서 / 어디서 막히는가" + 근거 URL
3. **Problem Statement**: 1개 선정 + 선정 이유 (문항 2 초안의 뼈대)
4. **Opportunity Sizing**: 공개 숫자 기반 대략치 (출처 필수)
5. **→ 플러그인 결정**: 이 문제를 어떤 클러치 피처로 푸는가 (한 문장 — 문항 1 초안의 뼈대)

## 도구

- Claude Code (WebSearch / WebFetch / 딥 리서치), NotebookLM(영상 분석)
- **어떤 도구를 쓰든 그 대화도 로그 제출 대상** — 훅이 없는 도구(NotebookLM·ChatGPT 웹 등)의 대화는 원본 그대로 텍스트로 내보내 해당 회사 `logs/`에 넣는다.
- AI 채팅에 비밀정보(API 키·토큰·비밀번호) 입력 금지.
