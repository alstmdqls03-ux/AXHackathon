## 0. 핵심 요약

- **과제**: 선택한 기업(그 기업·산업·고객)이 겪는 **실제 문제를 공개 자료로 입증**하고, 이를 해결하는 **Codex 플러그인** 제출. **[확정]**
- **마감**: **2026-07-10(금) 23:59:59** — 마감 후 제출 불가, 마감 전까지는 제출 후에도 계속 수정 가능. **[확정]**
- **제출물**: 기업별로 (1) submission.zip (≤100MB) + (2) 질문 5문항 답변(전 문항 필수) + (3) 출처 URL — 출처 URL은 규정상 **선택**이나 공개 자료 입증 요건상 사실상 필수. **[확정]**
- **기업 여러 개 선택 시 기업마다 별도 플러그인·별도 제출.** 하나의 제출물로 여러 기업 불가. 일부 기업만 제출해도 불이익 없음. **[확정]**
- 현재 계정에 표시된 제출 대상 기업: **채널톡 / 카카오페이증권 / 삼일PwC**

## 1. submission.zip 구성

과제 페이지에 명시된 트리 **[확정]** (트리 자체는 원문 그대로, 주석의 필수/선택 판정은 아래 표의 본문 규칙에서 도출):

```
submission.zip
├── src/                              # 필수 — 전체 플러그인 루트가 src 안에 있어야 함
│   ├── .codex-plugin/plugin.json     # 필수
│   ├── skills/<이름>/SKILL.md        # 조건부 필수(아래 참조) — 가장 일반적인 구성 요소
│   ├── .mcp.json                     # 선택 — MCP 서버 쓸 때
│   └── (그 밖의 실행 코드·설정 파일)  # 선택 — 자유 구조
├── README.md                         # 필수 — 문항 3의 세부 내용이 여기에도 적혀 있어야 함
└── logs/                             # 필수 — AI 대화 로그 전체
```

| 항목 | 필수 여부 | 근거 |
|---|---|---|
| `src/` (플러그인 루트) | **필수** | "전체 플러그인 루트가 src 안에 있어야 합니다" **[확정]** |
| `src/.codex-plugin/plugin.json` | **필수** | "src/.codex-plugin/plugin.json은 필수입니다" **[확정]** |
| plugin.json 외 동작 구성 요소 ≥ 1개 | **필수** | "플러그인이 실제로 동작하도록 만드는 구성 요소를 하나 이상" — 스킬(SKILL.md)이 가장 일반적, MCP 서버(.mcp.json)·직접 작성한 실행 코드도 가능 **[확정]** |
| `src/` 내부 그 외 구조 | 자유 | 단, **Codex 플러그인으로 실행이 되어야 함** **[확정]** |
| `README.md` (zip 루트) | **필수** | 트리에 명시 + 제출 폼 문항 3: "세부 내용은 zip 파일의 README.md에도 적혀 있어야 합니다" **[확정]** |
| `logs/` (zip 루트) | **필수** | 트리에 명시 + "대화 로그를 logs 폴더에 넣어 함께 제출합니다" **[확정]** |

- **zip 전체 최대 100MB, 확장자 .zip만 업로드 가능.** **[확정]** (제출 폼 + 운영 규정 4항)
- 운영 규정 4항: 제출물은 "빌드된 프로덕트"로서 **"평가 시 직접 구동될 수 있는 형태"**여야 함. **[확정]**
- **[권고/해석]** zip 루트에는 위 3개(src/, README.md, logs/) 외 다른 파일을 넣지 않는 것이 안전. 이 spec 파일 등 작업 문서는 zip에 포함하지 말 것.
- **[권고/해석]** "직접 구동" 요건 대비: 심사자가 zip을 받아 로컬 marketplace에 등록하면 바로 도는 상태를 목표로. OpenAI 문서 기준 로컬 설치는 repo 방식(`$REPO_ROOT/plugins/`에 복사 + `$REPO_ROOT/.agents/plugins/marketplace.json` 등록) 또는 personal 방식(`~/.codex/plugins/`에 복사 + `~/.agents/plugins/marketplace.json` 등록) → Codex 재시작. README.md에 설치·실행 절차를 그대로 적어두면 심사자가 헤매지 않음.

## 2. 플러그인 필수 파일 상세

### src/.codex-plugin/plugin.json — 필수 **[확정]**

해커톤 규정은 "필수"라는 사실만 명시. 파일 스키마는 OpenAI 공식 문서 기준 **[권고/해석 — OpenAI 문서 출처]**:

```json
{
  "name": "my-plugin-name",        // 필수, kebab-case
  "version": "1.0.0",              // 필수
  "description": "한 줄 설명",      // 필수
  "skills": "./skills/"            // 선택 — 경로는 플러그인 루트 기준 "./" 상대경로
}
```

- 필수 필드: `name`, `version`, `description`
- 선택 필드: `author`, `homepage`, `repository`, `license`, `skills`, `mcpServers`, `apps`, `hooks`, `interface`
- 참고 문서 (과제 페이지가 직접 링크) **[확정]**:
  - 플러그인 개요: https://developers.openai.com/codex/plugins
  - 플러그인 만들기: https://developers.openai.com/codex/plugins/build
  - 스킬 작성법: https://developers.openai.com/codex/skills

### 동작 구성 요소 (하나 이상 필수) **[확정]**

| 구성 요소 | 위치 | 비고 |
|---|---|---|
| 스킬 | `src/skills/<이름>/SKILL.md` | "가장 일반적" — 과제 페이지 표현 |
| MCP 서버 | `src/.mcp.json` | 필요 시 |
| 실행 코드 | `src/` 자유 위치 | 직접 작성한 코드·설정 |

## 3. 로그 요건 (logs/)

### 규정 **[확정]**

1. 플러그인을 만들면서 **AI와 주고받은 대화 로그**를 `logs/` 폴더에 넣어 제출.
2. **편집·발췌 없이 전체를 그대로** 제출.
3. 허용 형식: **md, txt, json, jsonl**. 이미지·PDF 등 텍스트 추출 불가 형식은 **미제출로 간주** (원문은 수동 저장 도구 안내 맥락에 등장 — 로그 전체에 적용된다고 보는 것이 안전 **[해석]**).
4. **⚠️ 실격**: 로그 파일을 원본 그대로가 아닌 **사후 가공(편집·발췌·삭제)** 하여 제출 시 실격. 운영 규정 7항도 "제출 후 프롬프트·과정 로그의 사후 가공·조작"을 실격 사유로 명시.
5. **로그·플러그인·질문지(5문항) 사이의 정합성**을 함께 평가 — 서로 어긋나면 불리.
6. AI 채팅에 **API 키·비밀번호·토큰 등 비밀정보 입력 금지** — 로그에 그대로 남음.

### 수집 방법 **[확정]**

- **Claude Code · Codex CLI**: log-hooks.zip( https://github.com/jocoding-ax-partners/axwar/releases/download/v1.0/log-hooks.zip )을 **작업 폴더 루트에 압축 해제** → 훅이 자동 저장. Codex CLI는 최초 실행 시 훅 신뢰 여부를 물으면 승인.
- **그 밖의 도구(Cursor·ChatGPT 웹·Codex 앱(GUI) 등)**: 전체 대화를 편집·발췌 없이 텍스트 형식으로 저장해 logs 폴더에 직접 넣기. Codex 앱은 훅 자동 수집 대상이 아님. **[운영진 답변]**

### 훅 동작 방식 (v1.0 코드 직접 확인) — 참고 정보

- 구성: `.claude/settings.json`(Claude Code용 Stop+SessionEnd 훅), `.codex/hooks.json`(Codex CLI용 Stop 훅), `tools/save_log.py`
- 매 턴 종료 시 transcript를 읽어 **`logs/claude-code/<session_id>.jsonl`** 또는 **`logs/codex/<session_id>.jsonl`** 로 저장.
- save_log.py는 사용자/어시스턴트 **대화 텍스트 라인만 남기고** tool_use·tool_result·thinking·메타 라인은 제외(주최측 도구의 공식 동작). 파싱 실패 시 원본 전체 복사로 폴백.
- **[권고/해석]** "원본 그대로"의 기준 = **훅이 생성한 파일에 손대지 않는 것**. 훅 출력물이 대화 텍스트만 담는 것은 주최측 도구의 설계이므로 문제 없음. 참가자가 그 파일을 열어 수정·삭제·발췌하는 순간 실격 리스크.

### 운영진 답변으로 확정된 로그 관련 사항 **[운영진 답변]**

- 훅은 **작업 시작 전 설치가 전제** (실시간 저장 도구이므로).
- 훅 설치 전에 진행한 작업은 **처음부터 다시 할 필요 없음** — 해당 대화의 원본만 텍스트로 확보해 제출하면 됨.
- 수동 내보내기에 메타데이터가 대량 포함되는 경우: 운영진은 가부를 확답하지 않음 — "정답은 없습니다. 참가자분께서 맞다고 생각하시는 방향으로 제출 부탁드립니다." → 원본 그대로 원칙이 우선이므로 메타데이터째 포함이 안전하다고 판단 **[권고/해석]**.
- 생성 시점에 참조된 지침 문서(에이전트 규칙 등)를 **모두 공유할 필요는 없으나**, **문제 정의·해결과 관련 있는 내용이면 포함하는 것이 채점에 불이익이 없음**.

### 이 저장소의 현재 상태 + 오케스트레이터 로그 관련

- ✅ 2026-07-09 log-hooks.zip을 저장소 루트에 압축 해제 완료 (`.claude/settings.json`, `.codex/hooks.json`, `tools/save_log.py`). **다음 Claude Code 세션부터** 훅이 `logs/claude-code/`에 자동 저장.
- ⚠️ 훅 설치 **이전** 세션(이 spec을 만든 세션 포함)은 자동 저장되지 않음 → 해당 세션 transcript를 편집 없이 md/txt/json/jsonl로 내보내 `logs/`에 넣을 것. **[운영진 답변 준용]**
- **오케스트레이터(서브에이전트/워크플로) 사용 시 [권고/해석]**: save_log.py는 훅 페이로드의 transcript_path(메인 세션 transcript)만 처리하므로 서브에이전트 대화는 저장되지 않는 것으로 보임 — 코드 분석 기반 추정이니 실제 사용 후 `logs/` 산출물에 포함 여부를 확인할 것. 규정은 "AI와 주고받은 대화 로그 전체"를 요구하고, 허용 형식(md/txt/json/jsonl) 내 추가 로그 제출을 금지하는 조항은 없음 → 오케스트레이터·서브에이전트 로그를 **별도 하위 폴더(예: `logs/orchestrator/`)에 원본 그대로** 넣는 것이 규정 취지(투명성·정합성 평가)에 부합하고 안전. 단, 이 역시 **사후 가공 없이 원본 그대로**여야 함.

## 4. 질문 5문항 (제출 폼 기준, 전 문항 필수*)

| # | 질문 | 폼 안내문 (그대로) | 제한 |
|---|---|---|---|
| 1 | 무엇을, 누가, 어떤 상황에서 쓰나요? | 만든 플러그인을 한 문장으로 요약해 주세요. 이어서 선언한 기업의 누가, 어떤 상황에서, 어디서 막혀 이 플러그인을 쓰게 되는지 구체적으로 적어 주세요. | **800자** |
| 2 | 왜 이 문제를 선택했나요? | 이 문제를 고른 이유와, 그 기업의 직원이나 고객, 또는 해당 산업의 사용자가 실제로 어디서 왜 막히는지 적어 주세요. | **800자** + **출처(URL) 입력란** (https:// 형식, +버튼으로 여러 개 추가 가능) |
| 3 | 플러그인은 어떻게 작동하나요? | 플러그인이 어떻게 동작하는지, 절차, 지식, 판단 기준을 어떻게 가져가는지 적어 주세요. 정보가 부족하거나 잘 안 풀리는 상황에서의 동작도 함께 적어 주세요. **세부 내용은 zip 파일의 README.md에도 적혀 있어야 합니다.** | **1000자** |
| 4 | AI를 어떻게 썼나요? | AI에게 맡긴 작업과 직접 판단한 부분을 구분해 적어 주세요. 만들면서 막혔던 지점, 해결 과정, 받아들이지 않은 AI 제안과 그 이유도 적어 주세요. | **800자** |
| 5 | 어떻게 검증했나요? | 입력에서 결과로 이어지는 예시를 하나 이상 적어 주세요. 정상 상황과 예외 상황을 어떻게 확인했는지, 무엇을 의심했고 무엇이 아직 부족한지, 테스트하며 고친 점을 적어 주세요. | **800자** |

- 출처 URL 필드는 운영 규정 4항에서 "출처 URL(선택)"으로 표기 — 폼에서도 * 없음. 단 과제 본문의 "실제 문제를 **공개 자료로 입증**" 요건과 "출처는 공개 자료로 AI가 확인할 수 있어야" 규정 때문에 **사실상 반드시 채워야 하는 항목**. **[권고/해석]**
- **[권고/해석]** 문항 5는 로그에 실제 검증 과정이 남아있어야 정합성 평가에서 안전 — 답변에 적은 테스트는 로그에서 재확인 가능해야 함.
- 'AI를 활용해 창출했던 작은 성과·경험'(공식 홈페이지 선발 여정 문구)은 **예선 제출물에 포함하지 않아도 됨** — 상위 200~300명에게 추후 별도로 전달받을 예정. **[운영진 답변]**

### 제출 절차 **[확정 — FAQ Q3]**

1. 과제 화면 상단의 **기업 버튼**(가입 시 선택한 기업만 표시) 클릭 → 해당 기업의 답변 등록 화면.
2. 각 문항 작성 + 산출물 ZIP 첨부.
3. 모든 필수 항목(*) 작성 후 제출 버튼.
- 폼 URL 패턴: `/rounds/10/opportunities/{id}/submission/new` (채널톡=1, 카카오페이증권=4, 삼일PwC=5)
- **마감 전까지 제출 후에도 계속 보완(수정) 가능.** **[운영진 답변 + 운영 규정 3·4항]**
- ⚠️ **참가기업 변경 시**: 제외되는 기업에 이미 제출한 해답이 있으면 **함께 삭제됨**. **[확정 — FAQ Q4]**

## 5. 실격·주의 규정

### 실격 사유 (운영 규정 7항 전체) **[확정]**

1. **표절·도용**
2. **대리 응시**
3. **부정행위**
4. **제출 후 프롬프트·과정 로그의 사후 가공·조작** (과제 페이지: 편집·발췌·삭제 포함)
5. **불법·비방 콘텐츠**
- 추가: 연령 미달(만 19세 미만)·허위 정보 시 실격·크레딧 회수 (규정 1항). 1인 1계정·본인 명의. **[확정]**

### 근거(출처) 요건 **[확정]**

- 출처는 **공개 자료**로 **AI가 확인할 수 있어야** 함.
- 인정되지 않는 근거: **기업 내부의 비공개 정보 / 확인할 수 없는 사적인 경험 / 출처 없는 숫자**.

### 비밀정보 **[확정]**

- AI 채팅에 **API 키·비밀번호·토큰 등 비밀정보 입력 금지** (로그에 남음).
- 제출물 적법성 보증 (규정 5항): 타인의 개인정보·기밀·오픈소스 라이선스 위반·제3자 API 키 등 권리침해 요소 미포함을 **참가자가 보증**.

### 기타 의무 **[확정]**

- **AI 사용 필수**, 사용 과정을 투명하게 공개 (규정 6항).
- 제출한 예선 제출물은 **본선 심사에 활용될 수 있음** (과제 페이지 ❗️ + FAQ Q8: 본선에서 예선·본선 답변을 함께 평가).

### 제출물의 취급·IP (참고) **[확정]**

- 풀이 결과·제출물은 선택한 기업에 제공됨 — 가입 시 **제3자 제공 동의** (규정 2항). 공개를 원치 않는 내용은 애초에 제출물·로그에 넣지 말 것.
- 저작권은 **참가자 귀속**. 주최·참여기업은 심사·홍보·평가 피드백(비공개 검토) 목적의 비독점·무상 사용권만 가짐. 실서비스 활용·상업화는 별도 라이선스 협의 (규정 8항).
- 코드·로그·설계자료는 **채용·심사·평가 목적으로만** 기업에 제공(동의서 징구), 행사 종료 시 사본 포함 파기, 과정 로그·비공개 설계자료는 홍보 목적으로 공개되지 않음 (FAQ Q6).
- 아이디어·문제 인식 자체는 법적 독점 대상이 아니며, 결과물 유사성만으로는 도용으로 보지 않음 (규정 9항 — 실격 사유 '표절·도용'의 판단 기준과 연결).
- 참여 기업이 제공한 데이터·내부 자료는 해커톤 참가 목적으로만 사용, 행사 종료 시 파기 (규정 10항).

## 6. 채점 관련 **[확정]**

- 예선 과제는 **AI와 심사자가 함께 평가**.
- **로그·플러그인·질문지 사이 정합성**을 함께 평가 — 어긋나면 불리.
- 문제 정의 자체가 참가자의 몫 (FAQ Q1: 운영진은 힌트·정답을 주지 않음. 기업 인터뷰 영상이 참고자료).
- 기업 인터뷰 영상 속 문제는 **본선 출제 문제**이며, 이를 활용해 **예선 답변을 제출해도 무방** (FAQ Q8). 영상은 조코딩·조코딩AX파트너스 채널에 6/24~7/7 순차 공개 — 채널톡·삼일PwC·카카오페이증권 편 모두 공개 완료 (FAQ Q2 + 공지). **[확정]**
- **[권고/해석]** "AI가 평가"하므로: 출처 URL은 로그인 없이 접근 가능한 공개 페이지로, README는 기계가 읽기 좋은 구조로, 플러그인은 문서만 보고 실행 가능하게.
