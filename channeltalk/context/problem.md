## 공통 컨텍스트 (요약)

채널톡은 "고객과 기업의 커뮤니케이션 문제를 푸는 회사"[E01, 00:18]로, 현재 명시적으로 "**이커머스 고객사의 상담 문제**"를 AI 에이전트로 풀고 있다[E02, 01:28]. 그 제품이 ALF: Knowledge/Rules/Tasks 구조[E10], 3,000여 기업·누적 650만 건 상담[E14]. 공개 사례의 AI 해결률은 55~80% 스펙트럼(안다르 61%[E15], 베타 최고 80%[E12], 라룸 배송문의 55%[E25]). 상세: `company.md`.

## 후보 A — ALF 해결률 갭: 미해결 상담 → 지식 보강 사이클이 수작업

### 1. 문제 정의
- **누가**: 이커머스 고객사의 CX 담당자(대개 1~4명 소규모 팀[E16, E20]) + 이를 3,000여 고객사[E14]에 반복 지원해야 하는 채널톡 측 AX 컨설턴트
- **어떤 상황에서**: ALF 도입 후에도 상담의 20~45%가 미해결로 상담사에게 이관된다(공개 해결률 55~80%의 여집합[E35(추정·파생), E12, E15, E25]). 해결률을 올리는 유일한 공개 경로는 지식(FAQ·도큐먼트)·규칙 보강이다 — ALF는 이전 상담 내역을 학습하지 않고 등록된 지식 기반으로만 답한다[E17].
- **어디서·왜 막히는가**: 미해결 로그 수천 건을 사람이 읽고 → 패턴을 분류하고 → 빠진 지식을 찾아 FAQ를 쓰고 → 품질을 검증하는 전 과정이 수작업. 최상 조건(자료 사전 정리)에서도 FAQ 65개 등록에 2.5시간[E16]이며, 그 앞단(로그에서 갭 도출)은 훨씬 크다. 이 사이클의 자동화 발표는 공개 검색에서 미발견[E36(추정)].

### 2. 근거 (출제자 힌트 최우선)
- **힌트 영상**: "이커머스 고객사의 상담 문제를 풀고 있어요"[E02, **01:28**] — 영상에서 유일하게 명시된 현재 문제의 한복판. "AI가 작업은 다 해주고 사람이 하는 건 생각"[E03, **02:16**] — 갭 판단·지식 설계가 정확히 그 "생각" 노동. "수천 개, 수만 개 기업의 문제를 풀 수 있는 게 AX 컨설턴트의 가장 귀중한 경험"[E06, **03:47**] — 고객사마다 반복되는 구조와 일치.
- **대장**: 해결률 61%[E15]·80%[E12]·55%[E25] / FAQ 품질=답변 품질(공식 가이드)[E17] / 세팅 단가[E16] / 자동화 공백[E36]

### 3. Opportunity Sizing
- 분석 대상 규모: 누적 650만 건[E14] × 미해결률 20~45%[E35] ≈ **130만~293만 건**(추정: 산술 여집합 적용)
- 고객사당: 자란다급(일 1,500건[E20])이면 미해결 일 300~675건(추정: 1,500×0.20~0.45), 월 ~9,000~20,000건이 사람 몫
- 노동 단가: FAQ 65개 등록 = 2.5h(최상 조건)[E16] → 로그 분석·갭 도출·검증 포함 시 고객사당 수 주(추정: E16이 하한) — **1개 고객사 1사이클 ≈ 사람 ~2주 스코프**
- 반복 배수: ALF 고객사 3,000여 개[E14]가 각자 반복 → 컨설턴트 관점 연중 상시 업무[E06]

### 4. 플러그인 적합성
- **~2주 업무인가**: 그렇다 — 위 3절 계산 (추정)
- **Codex 플러그인 데모 가능한가**: 그렇다 — 입력(상담 로그 CSV/텍스트 + 기존 FAQ) → 출력(갭 리포트 + FAQ/규칙 초안 + 재시뮬레이션 검증 리포트)이 전부 로컬 파일 파이프라인. 외부 API·비밀키 불요, 합성/공개 데이터로 정상·예외 실행 시연 가능
- **마감 내(2026-07-10) 구현 가능한가**: 그렇다 — 분류→갭→생성→검증 4단계 스킬로 최소 버전 구성 가능
- **회사 기존 제품과 중복 없나**: 없다 — 'AI 상담 시뮬레이션'[E13]은 도입 **전** 데모용, 본 후보는 도입 **후** 개선 사이클(방향은 같아 회사 로드맵 정합성도 있음)

## 후보 B — 신규 고객사 ALF 온보딩(지식 초기 구축)이 느리다

### 1. 문제 정의
- **누가**: ALF를 새로 도입하는 이커머스 고객사와 온보딩을 돕는 채널톡 측
- **어떤 상황에서**: 사이트·정책 문서·기존 상담 내역에 흩어진 정보를 Knowledge/Rules/Tasks[E10]로 정리해야 ALF가 작동
- **어디서 막히는가**: 자료 수집·정리·등록이 수작업 — 최상 조건에서도 FAQ 65개=2.5h[E16], 자료가 정리 안 된 보통 고객사는 그 몇 배(추정)

### 2. 근거
- **힌트 영상**: "이커머스 고객사의 상담 문제"[E02, **01:28**]의 진입 단계
- **대장**: 세팅 노동[E16], FAQ 품질 좌우[E17], 신규 도입 증가세[E11→E14]

### 3. Opportunity Sizing
- 신규 도입: 2,000개사(2025-11)[E11] → 3,000개사(2026-05)[E14] = 6개월간 +1,000개사, **월 ~167개사 신규 온보딩**(추정: 균등 분포 가정)
- 개사당 세팅: 수 시간~수 일[E16 하한 + 자료 비정리 가중(추정)]

### 4. 플러그인 적합성
- 데모 가능(파일 입출력)·마감 내 구현 가능은 후보 A와 동일
- **치명 감점**: 채널톡이 'AI 상담 시뮬레이션'(2026-05-27, 웹 크롤링+RAG 자동 지식화)[E13]으로 **이미 자동화에 착수한 정면 영역** — "회사가 못 푼 문제"가 아니어서 문제 선정의 설득력(왜 이 문제인가[E03, 01:36])이 약함

## 후보 C — 장바구니 이탈 고객 AI 아웃바운드 리커버리 설계

### 1. 문제 정의
- **누가**: 이커머스 운영자(마케팅/CRM 담당)
- **어떤 상황에서**: 평균 70.22%의 장바구니가 이탈[E40]하는데 회수 액션(왜 안 샀는지 인터뷰, 쿠폰 유도)은 설계·실행 비용이 큼
- **어디서 막히는가**: 이탈 사유 파악과 개인화 리커버리 캠페인 설계가 수작업

### 2. 근거
- **힌트 영상**: "'장바구니 이탈한 고객 알려줘'라고 하면 바로 AI가 전화를 걸어 인터뷰… 쿠폰으로 구매 유도"[E05, **03:27**] — 출제자가 직접 그린 시나리오
- **대장**: 이탈률 70.22%[E40], 채널톡의 고객 접점 통합 자산[E18]

### 3. Opportunity Sizing
- 장바구니 이탈률 70.22%[E40] — 모수는 크나, **회수율·건당 가치에 대한 채널톡 측 공개 수치가 대장에 없음** → 기대 효과 정량화 불가(추정 불가를 명시)

### 4. 플러그인 적합성
- **치명 감점 1**: 힌트 영상 내 위계상 확장 예시 — "상담 **이외에** 있는… 문제들"[E05, 03:40] (해석 I5: 상담이 본진). 명시된 현재 문제는 후보 A/B의 도메인[E02, 01:28]
- **치명 감점 2**: AI 전화 아웃바운드는 전화 인프라·고객 DB 의존 → **Codex 플러그인 단독으로 실행·검증 데모 불가** (스크립트 생성까지만 가능하고 "실행"은 시연 못 함)

## 추천: 후보 A — "ALF 미해결 상담 → 지식 보강 사이클" 자동화

**추천 이유** (질문 문항 1·2 초안 뼈대):
1. **[문항 2: 왜 이 문제인가]** 출제자가 명시한 유일한 현재 문제("이커머스 고객사의 상담 문제"[01:28])의 성과 지표(해결률)를 직접 움직이는 병목이다. 공개 수치로 문제가 입증되고(해결률 55~80% → 미해결 20~45%[E12,E15,E25,E35]), 회사가 아직 자동화를 발표하지 않은 공백이며[E36], 3,000개 고객사마다 반복되는 "AX 컨설턴트의 일"[03:47]이라 스케일이 있다. 개선 수단(지식 보강)이 공식 가이드로 확인되는[E17] — 즉 해법의 레버가 실재한다.
2. **[문항 1: 무엇을 만드나]** 상담 로그와 기존 FAQ를 입력하면 ① 미해결 문의를 자동 분류하고 ② 지식 갭을 도출해 ③ 갭별 FAQ·규칙 초안을 생성한 뒤 ④ 그 지식으로 미해결 문의를 재시뮬레이션해 "해결률이 얼마나 오르는가"를 검증 리포트로 내놓는 Codex 플러그인 — 사람 ~2주의 개선 사이클 1회를 한 번의 실행으로 대체한다.
3. 평가 관점 정합: 분류·갭 도출·생성·검증 각 단계의 판단 기준과 프롬프트 설계가 그대로 드러나는 구조 — "프롬프팅·접근·판단을 본다"[E03, 02:12]에 부합.

**탈락 이유**:
- **후보 B**: 회사가 'AI 상담 시뮬레이션'[E13]으로 이미 착수한 영역과 정면 중복 — "왜 이 문제를 골랐는가"[01:36]에 답이 궁색해진다.
- **후보 C**: 힌트 영상 위계상 확장 예시이고("상담 이외에"[03:40]), 전화 인프라 의존으로 플러그인 단독 데모가 불가하며, 채널톡 측 공개 수치가 없어 sizing이 일반 통계에만 의존한다.
