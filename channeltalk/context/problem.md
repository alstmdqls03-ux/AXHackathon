# problem.md — 채널톡 문제 정의

> **출처 가중치 규칙**: 힌트 영상(https://www.youtube.com/watch?v=5iRf37Z8Wd4)은 **출제 기업이 직접 준 힌트**로 이 문서의 모든 판단에서 **최상위 가중치**를 갖는다. 문제 선정·해석이 다른 공개 자료와 충돌하면 **힌트 영상이 우선한다.**

## 1. 컨텍스트

- 채널톡(채널코퍼레이션)은 "고객과 기업의 커뮤니케이션 문제를 푸는 회사" — 힌트 영상 00:18 (최상위 근거)
- 현재 집중 문제: **이커머스 고객사의 상담 문제**를 AI 에이전트로 해결 — 힌트 영상 01:28 (최상위 근거)
- 그 제품이 AI 상담 에이전트 **ALF(알프)**: 단순 응답을 넘어 주문 변경·취소·교환·반품 업무 자동화. 구조는 지식(Knowledge)·규칙(Rules)·태스크(Tasks). 누적 2,000여 개 기업 도입, 130만 건 상담 처리(2025-11) → 3,000여 개 기업, 650만 건(2026-05). https://channel.io/ko/blog/articles/alf-v2-press-f03760de , https://www.aitimes.com/news/articleView.html?idxno=210975
- 채널톡 자체는 22개국 약 17만 기업이 사용하는 이커머스 상담 1위 솔루션. https://channel.io/us

## 2. 문제 후보

### 후보 A — ALF 해결률 갭: 미해결 상담 로그 → 지식/규칙 개선이 사람 손에 달려 있다
- **누가**: 이커머스 고객사의 CX 담당자(대개 1~4명 소규모) + 채널톡 AX 컨설턴트
- **어떤 상황에서**: ALF 도입 후 해결률이 61%(안다르) ~ 80%(베타 최고치) — 즉 **20~39%는 여전히 미해결/상담사 이관**. 해결률을 올리려면 미해결 상담 로그를 사람이 읽고, 빠진 FAQ/지식을 찾아 작성하고, 규칙을 다듬고, 다시 테스트해야 한다.
- **어디서 막히는가**: 상담 로그는 수천~수만 건인데 갭 분석·FAQ 작성·검증은 전부 수작업. 베리시 사례에서 FAQ 65개 등록에만 2.5시간(자료가 사전 정리된 최상 조건) — 로그에서 갭을 찾아내는 앞단은 훨씬 크다. 이 사이클을 고객사마다 반복해야 한다.
- 근거: https://channel.io/ko/blog/articles/채널톡-안다르에-AI-에이전트-알프-공급-AI-상담-해결률-61-달성-59cc17fb (해결률 61%) / https://channel.io/ko/blog/articles/alf-v2-press-f03760de (해결률 80%) / https://channel.io/ko/blog/articles/c82e9aa3 (베리시 FAQ 세팅) / https://channel.io/ko/blog/articles/how-to-faq-7e2a6695 (FAQ 설정이 답변 품질을 좌우)

### 후보 B — 신규 고객사 ALF 온보딩(지식 구축) 자동화
- **누가**: ALF 도입을 검토·시작하는 이커머스 고객사
- **어디서 막히는가**: 사이트·정책문서·기존 상담 내역에 흩어진 정보를 Knowledge/Rules/Tasks로 정리하는 초기 세팅.
- **감점 요인**: 채널톡이 2026-05 'AI 상담 시뮬레이션'(웹 크롤링 RAG로 자동 데모)을 이미 출시 — 회사가 자체 해결 중인 영역과 정면 중복. https://www.aitimes.com/news/articleView.html?idxno=210975

### 후보 C — 장바구니 이탈 고객 AI 아웃바운드 캠페인 설계
- 힌트 영상 03:27에서 직접 언급된 시나리오(AI가 전화로 이탈 사유 인터뷰, 쿠폰 유도).
- **감점 요인**: 영상에서 "상담 이외에" 확장 가능성의 예시로 언급된 것이고, 명시된 현재 문제는 "이커머스 고객사의 상담 문제"(01:28). 전화 아웃바운드는 플러그인 단독으로 실행·검증 가능한 형태로 만들기 어렵다(외부 인프라 의존).

## 3. Problem Statement (선정: 후보 A)

**이커머스 고객사가 ALF(AI 상담 에이전트)를 도입해도 상담의 20~39%는 미해결로 남는데, 그 원인인 지식 갭을 찾아 FAQ·규칙을 보강하고 재검증하는 사이클이 전부 사람의 수작업이다.** 상담 로그 수천 건을 읽고 → 미해결 패턴을 분류하고 → 빠진 지식을 찾아 FAQ 초안을 쓰고 → 테스트하는 이 일은 CX 담당자/AX 컨설턴트가 고객사마다 반복하는 ~2주 분량의 업무다.

선정 이유:
1. 힌트 영상이 명시한 현재 문제("이커머스 고객사의 상담 문제", 01:28)의 한복판이다.
2. 힌트 영상의 AX 컨설턴트상("수천 개, 수만 개 기업의 문제를 푸는 경험", 03:47)과 일치 — 고객사마다 반복되는 컨설팅 업무를 플러그인화한다.
3. 회사가 이미 푼 영역(온보딩 데모=시뮬레이션 기능)과 겹치지 않고, 도입 **후** 운영 단계의 갭이라 공개 수치(해결률 61~80%)로 문제 존재가 입증된다.
4. 공개 데이터(상담 로그 형식, FAQ 구조)만으로 플러그인이 직접 구동·검증 가능하다.

## 4. Opportunity Sizing

- ALF 도입 기업 3,000여 개 × 미해결률 20~39% (해결률 61~80%의 여집합) — 고객사마다 반복되는 개선 사이클. https://www.aitimes.com/news/articleView.html?idxno=210975
- 규모 예시: 자란다 일 문의 ~1,500건, 라룸 일 300~500건 — 미해결분만 월 수천 건의 분석 대상. https://channel.io/ko/blog/articles/4c6c135e
- FAQ 세팅 단가: 정리된 자료 기준 FAQ 65개 = 2.5시간(베리시). 로그 분석~갭 도출까지 포함하면 고객사당 수 주. https://channel.io/ko/blog/articles/c82e9aa3

## 5. → 플러그인 결정 (초안 — 사용자와 논의 후 확정)

**상담 로그(CSV/텍스트)를 넣으면 미해결 문의를 자동 분류해 지식 갭을 찾고, 갭별 FAQ·규칙 초안을 생성한 뒤, 생성된 지식으로 미해결 문의를 재시뮬레이션해 "해결률이 얼마나 오르는지"까지 검증 리포트로 내놓는 Codex 플러그인** — "AI 상담 해결률 개선 컨설턴트" 한 사이클(~2주 업무)을 한 번의 실행으로 대체한다.

- 힌트 영상 평가 기준과의 정합: 왜 이 문제인가(01:36) → 위 3절 / 판단·프롬프팅(02:12) → 분류·생성·검증 각 단계의 프롬프트 설계가 곧 산출물.
