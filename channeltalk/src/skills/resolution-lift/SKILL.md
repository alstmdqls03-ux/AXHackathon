# resolution-lift — AI 상담 해결률 개선 사이클

이커머스 AI 상담(예: 채널톡 ALF)은 등록된 지식(FAQ·문서) 기반으로만 답한다. 그래서 해결률을 올리는 유일한 레버는 **미해결 상담에서 지식 갭을 찾아 FAQ를 보강하는 것**인데, 이 사이클(로그 읽기 → 갭 찾기 → FAQ 쓰기 → 검증)이 수작업이다. 이 스킬은 그 1사이클을 대체한다.

**분업 원칙**: 세는 일(집계·검증·리포트 조립)은 스크립트가, 생각하는 일(분류·초안·채점)은 네(LLM)가, **정책 결정은 사람이** 한다. 네 몫의 판단 기준은 아래에 전부 명문화되어 있다 — 임의로 바꾸지 마라.

## 입력

| 파일 | 필수 컬럼 | 설명 |
|---|---|---|
| tickets.csv | `ticket_id`, `question` (권장: `alf_answer`, `resolved`, `agent_answer`) | 상담 로그. `resolved=false`가 미해결(상담사 이관) 건 |
| faq.csv | `question`, `answer` | 현재 등록된 지식 |

사용자가 파일을 주지 않으면 위 형식을 안내하고 경로를 물어라. 체험용으로는 플러그인 동봉 `examples/tickets.csv`, `examples/faq.csv`를 쓴다.

## 절차 (4단계 — 순서 고정)

작업 디렉터리는 `out/`(또는 사용자 지정). `<plugin>`은 이 플러그인의 루트 경로다.

### 1단계 — 적재·검증 (스크립트, 결정적)

```bash
python3 <plugin>/scripts/load.py <tickets.csv> <faq.csv> --out out
```

성공 시 `out/summary.json`(집계), `out/unresolved.json`(미해결 목록), `out/faq.json`(기존 지식). **실패(exit 1) 시 스크립트가 출력한 안내(필수 컬럼·템플릿)를 사용자에게 그대로 전달하고 중단하라 — 컬럼을 짐작해서 진행하지 마라.**

### 2단계 — 갭 분류 (너의 판단)

`out/unresolved.json`의 모든 티켓을 읽고, 주제가 같은 티켓끼리 클러스터로 묶은 뒤 클러스터마다 갭 유형을 판정해 `out/gaps.json`에 저장하라.

**갭 유형 판정 기준** (이 순서로 검사, 첫 매치 적용):

| 유형 | 정의 | 판정 신호 |
|---|---|---|
| **D 태스크 필요** | 지식으로 해결 불가 — 계정·주문에 대한 *실행*(조회·변경)이 필요 | 상담사 답변이 "~처리해 드렸습니다" 등 행위 완료형 |
| **C 정책 결정 필요** | 답의 근거가 되는 회사 방침이 로그에 없음 — 사람만 정할 수 있음 | `agent_answer`가 비어 있고 FAQ에도 근거 없음 |
| **B 지식 불명확** | 관련 FAQ가 *있으나* 예외·조건이 빠져 답변 실패 | `out/faq.json`에 같은 주제 항목 존재 |
| **A 지식 부재** | FAQ에 해당 주제가 아예 없음, 상담사 답변에 정답이 있음 | FAQ 무관련 + `agent_answer` 있음 |
| **REVIEW 사람 검토** | 위 어디에도 확신 있게 넣을 수 없음 (감정 표현, 문의 아님, 맥락 부족) | — |

**confidence 기준**: 높음 = 근거 티켓 3건 이상이고 유형 판정에 반례 없음 / 중간 = 근거 1~2건 또는 경계 사례 / 낮음 = 판정은 했으나 반례 존재. **확신이 없으면 유형을 짐작하지 말고 REVIEW로 보내라.**

`out/gaps.json` 형식 (모든 필드 필수, `evidence_tickets`는 실제 ticket_id만):

```json
[{"cluster": "상품 케어(세탁·보관)", "gap_type": "A", "confidence": "높음",
  "evidence_tickets": ["T-1034", "T-1035"], "rationale": "FAQ에 소재·세탁 항목이 없고 상담사 실답변에 정답 존재"}]
```

미해결 티켓은 **한 건도 빠짐없이** 정확히 하나의 클러스터에 배정하라. 미해결이 200건을 넘으면 50건씩 나눠 읽고 클러스터를 병합하라(같은 주제는 같은 cluster 이름 사용).

### 3단계 — FAQ·규칙 초안 (너의 작성)

`gap_type`이 **A 또는 B인 클러스터에만** FAQ 초안을 작성해 `out/faq_draft.json`에 저장하라. C·D·REVIEW는 초안을 만들지 마라 — 사람 몫이다(4단계 리포트가 자동으로 "사람 확인 필요" 목록에 올린다).

**작성 규칙**:
1. 답변의 1차 근거는 반드시 근거 티켓의 `agent_answer`(상담사 실답변)다. 근거를 `basis_tickets`에 기록하라.
2. 상담사 답변들이 서로 모순되면 그 클러스터는 초안을 만들지 말고 gaps.json에서 REVIEW로 재분류하라.
3. `agent_answer` 없이 일반 지식으로 답을 쓸 수밖에 없다면 **반드시 `"policy_check": true`** — 리포트에 [정책 확인 필요]로 표시되고 등록이 보류된다. **회사 정책을 지어내지 마라.**
4. `question_variants`는 고객이 실제로 물을 법한 표현으로 3개 (근거 티켓의 원문 표현을 활용).
5. B 유형은 기존 FAQ 항목을 대체하는 게 아니라 **빠진 예외·조건을 추가**하는 답변으로 쓰라.

```json
[{"cluster": "상품 케어(세탁·보관)",
  "question_variants": ["원피스 세탁 어떻게 하나요?", "니트 보풀 관리법 있나요?", "청바지 물빠짐 원래 그런가요?"],
  "answer": "소재별 케어 안내: …", "basis_tickets": ["T-1034", "T-1035"], "policy_check": false}]
```

### 4단계 — 재시뮬레이션 채점 (너의 채점) → 검증·리포트 (스크립트)

미해결 티켓 **전건**에 대해 "기존 FAQ + 3단계 초안이 모두 등록됐다면 AI 상담이 이 질문을 해결할 수 있는가"를 판정해 `out/simulation.json`에 저장하라.

**채점 기준**:
- **해결**: 초안/기존 FAQ의 답이 질문에 직접·완결적으로 답함 (고객이 추가 문의 없이 종료할 수준)
- **부분**: 관련 정보는 주지만 개별 확인(주문 조회 등)이 남음
- **미해결**: 지식만으로 답 불가 (C·D·REVIEW 티켓은 대부분 여기 — 초안이 없으므로)
- 판정마다 어느 지식(기존 FAQ인지 초안 클러스터인지)이 근거인지 `reason`에 쓰라. **관대하게 채점하지 마라 — 이 숫자는 개선 기대치로 보고된다.**

```json
[{"ticket_id": "T-1034", "verdict": "해결", "reason": "'상품 케어' 초안이 세탁 방법을 직접 안내"}]
```

저장 후 실행:

```bash
python3 <plugin>/scripts/simulate.py out
```

스크립트가 스키마·환각 ID·전건 커버를 검증하고 통과 시 `gap-report.md`, `faq-draft.md`, `validation-report.md`, `report.html`을 생성한다. **검증 실패 시 오류 메시지가 가리키는 단계의 JSON을 고쳐 재실행하라 — 검증을 우회하거나 스크립트를 수정하지 마라.**

### 마무리 — 사용자 보고

리포트 생성 후 사용자에게 요약하라: ①기준 → 시뮬레이션 해결률(+상승분) ②상위 갭 클러스터 ③[정책 확인 필요]·C/D/REVIEW 등 **사람이 결정할 목록** ④초안은 검토 후 등록하라는 안내. 시뮬레이션 수치는 판정값이지 실측이 아님을 밝혀라.

## 정보 부족·실패 상황의 동작 (요약)

| 상황 | 동작 |
|---|---|
| 파일 없음/컬럼 누락 | load.py가 템플릿 출력 후 중단 — 짐작 진행 금지, 사용자에게 안내 |
| `resolved` 컬럼 없음 | agent_answer 존재로 추정하되 전 리포트에 (추정) 자동 표기 |
| 기존 FAQ 없음(빈 파일) | 전량 A(지식 부재) 기준으로 진행하고 온보딩 성격임을 보고에 명시 |
| 갭 판정 확신 없음 | REVIEW로 분류 — 초안 생성 안 함 |
| 답변 근거 없음 | policy_check=true 마커 — 정책을 지어내지 않음 |
| 상담사 답변 모순 | 해당 클러스터 REVIEW 재분류 |
| 시뮬레이션 개선 0 | 그대로 보고 + 갭 유형 분포에서 원인 가설 제시 |
| 미해결 200건 초과 | 50건 배치 분할 처리, 그 사실을 보고에 명시 |
| 로그 내 개인정보 | load.py가 전화·이메일 자동 마스킹(건수 리포트) |
